# 프로젝트 명 
 - 테트리스

# 목적
 - 한정된 공간 (컨테이너, 빠레트, 트럭 짐칸 등) 에 짐을 가장 효율적으로 싣기 위한 최적화된 짐 쌓기 (우리는 이를 테트리스로 부르도록 한다.) 가이드라인을 제공한다.
 - 즉, 최적화된 테트리스에 대한 가이드라인을 제공함으로써 효율적인 물류 효율을 도모한다.
 - 가장 최소한의 공간을 사용하여 필요한 짐을 실을 수 있도록 한다.

# 요구사항

## 짐쌓기 공간 : 공간
 - 한정된 공간이라는 개념을 "공간" 으로 명명한다.
 - 공간은 기본적으로 `파레트` `컨테이너` `2톤반 트럭 짐칸` 을 제공한다. 
 - 공간은 가장 메인으로는 파레트를 사용하고 기본 사이즈는 (1150 * 1150 * 1550mm) 로 한다. 
  - 나머지 공간에 대한 사이즈는 검색을 통해 지정해.
 - 공간은 가로/세로/높이 그리고 offset의 데이터를 가진다.
  - offset은 실제로 짐을 쌓을 때 가로/세로/높이에 대한 리미트 값이다.
  - 예를 들어, 빠레트의 가로/세로가 5cm / 5cm 이고, offset이 1cm / 1cm 라고 한다면, 해당 빠레트는 실제로 4cm / 4cm 안에 짐을 쌓을 수 있다.
 - 사용자는 기본으로 제공해주는 데이터를 직접 사용할 수 있지만, 직접 커스텀하여 공간을 추가할 수 있다.

## 짐 : 블록
 - 공간에 쌓을 짐을 "블록" 이라고 명명한다.
 - 각 블록 하나하나를 "블록 유형" 이라고 명명한다.
 - 가장 표준화된 박스의 기본적인 데이터를 사용해서 제공한다. 이 데이터는 가로/세로/높이 데이터를 가진다.
 - 마찬가지로, 블록은 사용자가 직접 커스텀하여 데이터를 추가할 수 있어야한다. 각 블록에는 이름을 명명할 수 있어야 한다.

## 테트리스 
 - 테트리스를 할 때, 어떤 공간에 쌓을 것인지 가장 먼저 선택한다. (파레트 or 컨테이너 ...)
 - 그 이후에 블록 유형과 각 블록 유형의 갯수를 입력받는다. 이 때, 해당 블록 유형을 결정하기 전에 체크박스로 fragile 여부를 받는다.
  - 만약, fragile 인 블록인 경우 테트리스 결과의 상단에 위치해야 한다.
 - 마지막으로 시작 버튼을 누르면 최적화 연산을 진행하고, 가장 최적화된 짐 쌓기 구조를 보여주어야 한다.
 - 여기서 최적화란, 가장 최소한의 공간을 사용하는 것을 의미한다. 단순하게 블록을 쌓는 것에서 끝나는 것이 아니라 안정적으로 흔들리지 않으며 올바르게 서있는 형태를 유지해야한다.
 - 각 블록은 어느 방향으로든 회전 가능하다.
 - 최적화 계산이 끝나면, 각 공간에 대한 3D 렌더링을 통해 어떤 박스가 어떻게 쌓여있는지 사용자가 확인할 수 있어야 한다.
 - 사용된 블록 유형이 하단에 나열되어 있고, 특정 블록 하나를 클릭할 경우 렌더링된 형태에서 해당 블록이 하이라이트되어야 한다. 
  - 다른 블록의 유형들은 투명하되, 윤곽선을 살려두어 전체적인 입체감은 살린다.

## 추가 블록 넣기
 - 최종 결과 화면에서 "추가 블록 넣기" 버튼에 대한 기능.
 - 해당 기능은 현재 최종 결과에서 특정 블록 유형을 선택하면, 해당 블록을 어디에 어떻게 쌓을 수 있을 지 결과를 알려주어야한다.
  - 즉, 최종 최적화 결과에서 비어있는 공간이 부피 단위로 15cm^3 이라고 해보자. 여기에 내가 1cm^3 블록 유형을 선택하면 총 15개의 블록이 쌓아진 결과화면이 보여야한다.
 - 이 기능은 계속해서 체이닝형태로 반복수행이 가능해야한다.
