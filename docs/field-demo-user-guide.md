# Package Tetris 현장 시연 가이드

## 1. 현재 시연 가능 여부

현재 버전은 현장 작업자가 PC나 태블릿 브라우저에서 시연해볼 수 있는 상태다.

다만 아직 정식 배포 서비스는 아니며, 아래 조건을 전제로 한다.

- 시연 PC에 Node.js와 npm을 설치해야 한다.
- 앱은 서버 계정 없이 해당 PC 브라우저에 작업 데이터를 저장한다.
- 여러 PC/태블릿 간 자동 동기화는 아직 지원하지 않는다.
- 팔레트, 20ft GP, 2.5톤반 preset의 대량 혼합 박스 자동 audit은 통과한다.
- 현장 PC에서는 `npm run field:audit`로 3개 현장형 시나리오 계산 시간과 안전 결과를 다시 확인할 수 있다.
- 실제 현장 투입 전에는 현장 데이터 추가 검증, 현장 PC 기준 대량 입력 성능 재측정, 현장 작업자 리허설이 필요하다.

현재 시연 대상 브랜치:

- `main`
- 시연 담당자는 GitHub 저장소의 `main` 브랜치 기준으로 프로젝트를 내려받으면 된다.

V1 역사 완료 기준과 운영 전 파일럿 확인 범위는 `docs/v1-readiness.md`를 기준으로 확인한다. V2 현행 결과 확인은 이 문서의 `3D와 공간 확인`, 미적재/경고, 백업 파일 순서를 따른다.

처음 실행하는 사용자는 먼저 `docs/non-developer-start-guide.md`를 확인한다. Windows에서 명령어 입력이 어렵거나 PowerShell `npm.ps1` 오류가 나오면 `docs/windows-cmd-launch-guide.md`와 `scripts/windows-start-package-tetris.cmd`를 사용한다.

## 2. 준비물

- Windows 10/11 또는 macOS PC
- Chrome 또는 Microsoft Edge 최신 버전
- 인터넷 연결
- 프로젝트 파일
- Node.js LTS

공식 기준:

- Node.js 공식 다운로드: https://nodejs.org/en/download
- npm 공식 설치 안내: https://docs.npmjs.com/downloading-and-installing-node-js-and-npm
- Next.js 실행 요구사항: https://nextjs.org/docs/pages/getting-started/installation
- Windows winget 설치 명령: https://learn.microsoft.com/windows/package-manager/winget/install
- Windows PowerShell 실행 정책 안내: https://learn.microsoft.com/powershell/module/microsoft.powershell.core/about/about_execution_policies
- Homebrew 설치 명령: https://brew.sh/
- Homebrew Node formula: https://formulae.brew.sh/formula/node
- Git 공식 다운로드: https://git-scm.com/downloads/win

Next.js 공식 문서 기준으로 Node.js는 최소 `20.9` 이상이 필요하다. 현장 시연 PC에는 Node.js 공식 사이트의 `LTS` 버전을 설치하는 것을 권장한다.

## 3. Node.js 설치

### Windows

권장 방식은 PowerShell 명령어 설치다.

1. Windows 시작 메뉴에서 `PowerShell`을 검색한다.
2. PowerShell을 연다.
3. 아래 명령어를 그대로 붙여넣고 Enter를 누른다.

```powershell
winget install -e --id OpenJS.NodeJS.LTS
```

4. 설치 중 동의 여부를 물으면 화면 안내에 따라 `Y`를 입력한다.
5. 설치가 끝나면 PowerShell을 모두 닫고 새로 연다.
6. 아래 명령어로 설치를 확인한다.

```powershell
node -v
npm -v
```

`node -v` 결과가 `v20.9.0` 이상이면 실행 가능하다.

`winget` 명령어를 찾을 수 없다고 나오면 Windows 업데이트를 먼저 진행하거나, Node.js 공식 다운로드 페이지에서 `LTS` Windows Installer를 받아 설치한다.

### macOS

명령어로 설치하려면 Homebrew를 사용한다. Homebrew가 이미 설치되어 있으면 2번부터 진행한다.

1. Terminal을 열고 아래 명령어로 Homebrew를 설치한다.

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

설치 중 비밀번호를 물으면 macOS 로그인 비밀번호를 입력한다. 비밀번호를 입력해도 화면에는 글자가 보이지 않을 수 있다.

2. Terminal을 닫고 새로 연 뒤 아래 명령어를 실행한다.

```bash
brew install node
```

3. 아래 명령어로 설치를 확인한다.

```bash
node -v
npm -v
```

`node -v` 결과가 `v20.9.0` 이상이면 실행 가능하다.

Homebrew 설치가 어렵거나 회사 보안 정책상 차단되면 Node.js 공식 다운로드 페이지에서 `LTS` macOS Installer를 받아 설치한다.

## 4. 프로젝트 파일 받기

방법은 두 가지다. 현장 작업자 PC에서는 `방법 A`가 더 쉽고, 담당자가 반복 시연 환경을 관리할 때는 `방법 B`가 더 편하다.

### 방법 A. ZIP 파일로 받기

1. GitHub 저장소에 접속한다.
   - https://github.com/inshakr2/Package-Tetris
2. 브랜치가 `main`인지 확인한다.
3. `Code` 버튼을 누른다.
4. `Download ZIP`을 누른다.
5. 다운로드한 ZIP 파일을 압축 해제한다.
6. 압축 해제한 폴더 이름이 너무 길면 `package-tetris`처럼 짧게 바꾼다.

### 방법 B. Git으로 받기

Git을 사용할 수 있는 담당자는 아래 명령어를 실행한다.

```bash
git clone -b main https://github.com/inshakr2/Package-Tetris.git
cd Package-Tetris
```

Windows에서 Git이 없다면 https://git-scm.com/downloads/win 에서 설치한다. 현장 작업자가 직접 Git을 다룰 필요는 없다.

Windows에서 Git도 명령어로 설치하려면 PowerShell에서 아래 명령어를 실행한다.

```powershell
winget install -e --id Git.Git
```

## 5. 앱 실행하기

아래 명령어는 프로젝트 폴더 안에서 실행해야 한다.

Windows에서는 압축 해제한 폴더를 연 뒤, 파일 탐색기 주소창에 `powershell`을 입력하면 해당 폴더에서 PowerShell이 열린다.

macOS에서는 Terminal에서 `cd` 명령어로 프로젝트 폴더로 이동한다.

### 5.1 처음 한 번만 실행

```bash
npm install
```

이 명령어는 앱 실행에 필요한 파일을 내려받는다. 인터넷 연결이 필요하며, PC에 따라 몇 분 걸릴 수 있다.

Windows PowerShell에서 `npm.ps1 파일을 로드할 수 없습니다` 또는 `이 시스템에서 스크립트를 실행할 수 없으므로` 오류가 나오면 PowerShell 실행 정책이 npm 실행 파일을 막은 것이다.

가장 간단한 우회 방법은 같은 프로젝트 폴더에서 명령 프롬프트를 사용하는 것이다.

```powershell
cmd
npm.cmd install
```

이후 실행도 같은 창에서 아래처럼 진행할 수 있다.

```cmd
npm run dev
```

PowerShell을 계속 사용해야 한다면 현재 사용자 범위만 허용하도록 아래 명령어를 실행한다. 관리자 권한은 필요 없다.

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

확인 문구가 나오면 `Y`를 입력한다. 이후 PowerShell을 닫고 새로 연 뒤 다시 실행한다.

```powershell
npm install
```

현재 적용된 정책을 확인하려면 아래 명령어를 사용한다.

```powershell
Get-ExecutionPolicy -List
```

회사 보안 정책 때문에 변경이 막히면 정책을 강제로 바꾸지 말고 `cmd`에서 `npm.cmd install`을 사용하거나 IT 담당자에게 Microsoft `about_Execution_Policies` 문서를 전달한다.

### 5.2 현장 PC audit

시연 담당자는 앱을 열기 전에 아래 명령어로 현재 PC 기준 적재 정합성과 계산 시간을 확인한다.

```bash
npm run field:audit
```

`Package Tetris 현장 audit 통과`가 보이면 시연용 preset 기준 계산은 통과한 것이다.

`Package Tetris 현장 audit 확인 필요`가 보이면 표시된 시나리오 이름을 개발 담당자에게 전달하고, 해당 PC에서 바로 시연하지 않는다.

시연 전 최종 확인을 한 번에 끝내려면 아래 명령어를 실행한다.

```bash
npm run v1:verify
```

이 명령어는 테스트, 타입 검사, 현장 audit, 빌드를 순서대로 실행한다. 중간에 멈추면 화면에 표시된 오류를 개발 담당자에게 전달한다.

### 5.3 시연용 실행

```bash
npm run dev
```

터미널에 `Local: http://localhost:3000` 또는 비슷한 문구가 보이면 실행된 것이다.

브라우저에서 아래 주소를 연다.

```text
http://localhost:3000
```

### 5.4 포트 충돌 시

이미 다른 프로그램이 3000번 포트를 쓰고 있으면 아래처럼 3001번으로 실행한다.

```bash
npm run dev -- --port 3001
```

브라우저에서 아래 주소를 연다.

```text
http://localhost:3001
```

### 5.5 실행 종료

터미널에서 아래 키를 누른다.

```text
Ctrl + C
```

종료 여부를 묻는 문구가 나오면 `Y`를 입력한다.

### 5.6 오프라인 준비 확인

앱을 한 번 연 뒤 상단의 저장 상태 버튼을 누르면 `작업 저장 상태` 패널이 열린다.

패널의 `오프라인 준비`가 `준비됨`으로 보이면, 인터넷 연결이 불안정해졌을 때 앱 화면을 다시 여는 데 도움이 된다.

같은 패널의 `앱 설치` 행에서는 현장 태블릿이나 PC에서 Package Tetris를 앱처럼 다시 여는 방법을 확인할 수 있다.

- Chrome 또는 Edge: `앱 설치` 버튼이 보이면 누른다. 버튼이 설치 창을 열지 않으면 주소창의 설치 아이콘이나 브라우저 메뉴의 앱 설치를 사용한다.
- Safari: 공유 버튼을 누른 뒤 `홈 화면 추가`를 선택한다.
- 설치 후에도 작업 데이터는 자동으로 다른 기기로 이동하지 않는다. 중요한 작업은 계속 백업 파일로 보관한다.

주의:

- 오프라인 준비는 백업 파일을 대체하지 않는다.
- 현재 작업 데이터는 같은 기기의 같은 브라우저에 자동저장되지만, 다른 PC나 다른 브라우저로 자동 이동하지 않는다.
- 시연 종료 전에는 반드시 `백업 파일 만들기`를 눌러 JSON 파일을 보관한다.
- 개발 시연에서는 `npm run dev` 터미널을 계속 실행해 둔다.

## 6. 현장 작업자 시연 순서

### 1단계. 적재 공간 선택

1. `공간 선택` 영역에서 기본 공간을 고른다.
2. 기본 제공 공간은 파레트, 컨테이너, `2.5톤반` 등이 있다.
3. 직접 공간을 만들려면 `내 공간 추가`를 누른다.
4. 폭, 깊이, 높이, 안전 여유를 입력하고 저장한다.

확인 포인트:

- 공간명이 현장 용어로 이해되는지 확인한다.
- `2.5톤반` 명칭이 현장 기준에 맞는지 확인한다.
- 안전 여유가 실제 작업 기준과 맞는지 확인한다.

### 2단계. 박스 등록

1. `박스 등록` 영역에서 박스 종류를 추가한다.
2. 박스명, 무게, 가로, 세로, 높이를 입력한다. 무게는 선택 입력이다.
3. 상위 그룹과 하위 그룹을 선택한다. 필요한 그룹이 없으면 `그룹 관리`에서 먼저 등록한다.
4. 깨짐주의 여부를 선택한다.
5. 자주 쓰는 박스는 저장된 박스로 등록해 재사용한다.
6. 저장된 박스가 많으면 `저장된 박스 찾아 추가`를 눌러 이름, 치수, 무게, 그룹으로 검색한 뒤 현재 작업에 추가한다.
7. 박스를 엑셀로 많이 등록해야 하면 먼저 `엑셀 포맷 보기`를 누른다.
8. 포맷 창에서 `샘플 파일 다운로드`를 눌러 `.xlsx` 샘플 파일을 받은 뒤, 필요한 값으로 수정한다.
9. `.xlsx` 파일 첫 번째 sheet에 아래 컬럼을 유지한다. 자동화 프로그램으로 파일을 만들 때도 열 이름과 순서를 바꾸지 않는다.

| 상위그룹 | 하위그룹 | 박스명 | 가로mm | 세로mm | 높이mm | 무게kg | 깨짐주의 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 금영 | 스피커 | K-스피커 포장 | 450 | 320 | 280 | 12.5 | Y |

10. 파일 준비가 끝나면 `엑셀로 박스 일괄등록` 또는 포맷 창의 `이 포맷으로 파일 선택`을 누른다.
11. 파일을 선택하면 바로 저장되지 않고 `엑셀 박스 일괄등록 미리보기`가 열린다.
12. 미리보기에서 `가져올 박스`와 `오류 행`을 확인한다. 오류가 있으면 행 번호와 사유를 보고 엑셀 파일을 고친 뒤 다시 선택한다.
13. 오류가 없거나 정상 행만 먼저 가져와도 괜찮으면 `일괄등록 적용`을 눌러 저장된 박스에 추가한다.
14. 빠르게 시연하려면 `현재 작업` 영역의 `시연 예제 불러오기`를 누른다. 이 버튼은 `2.5톤반` 기준 예제 박스를 현재 작업에 채운다.

확인 포인트:

- mm 단위 입력이 현장 작업자에게 자연스러운지 확인한다.
- 무게가 필수 입력처럼 보이지 않는지 확인한다.
- 상위 그룹과 하위 그룹을 직접 타이핑하지 않고 등록된 값에서 선택할 수 있는지 확인한다.
- 저장된 박스가 많아도 모달 검색으로 원하는 박스를 찾을 수 있는지 확인한다.
- `.xlsx` 일괄등록은 파일 선택만으로 즉시 저장되지 않고, 미리보기 확인 후 적용되는지 확인한다.
- 오류 행의 행 번호와 사유가 엑셀 파일 수정 행동으로 이어지는지 확인한다.
- 깨짐주의끼리 적층 허용 정책이 실제 업무 기준과 맞는지 확인한다.
- 시연 예제를 불러와도 기존 내 공간과 저장된 박스가 사라지지 않는지 확인한다.

### 3단계. 실행 전 확인

1. `실행 전 확인` 영역에서 박스 수량, 부피, 경고를 확인한다.
2. 무겁거나 먼저 바닥에 놓고 싶은 박스가 있으면 `이번 작업` 카드의 `아래층 우선`에서 `먼저 바닥에` 또는 `가장 먼저`를 선택한다.
3. `실행 전 확인`의 `하단 우선` 요약이 의도한 항목 수, 박스명, 수량과 맞는지 확인한다.
4. 문제가 없으면 `결과 만들기`를 누른다.
5. 실행이 막히면 화면에 표시되는 이유를 확인하고 입력값을 수정한다.

확인 포인트:

- “왜 결과를 만들 수 없는지”가 현장 작업자에게 이해되는지 확인한다.
- 경고 문구가 작업 행동으로 이어지는지 확인한다.
- 하단 우선 설정은 저장된 박스 원본이 아니라 이번 작업에만 적용되는지 확인한다.

### 4단계. 결과 확인

1. `결과 확인` 영역에서 사용 공간 수, 평균 적재율, 미적재 수를 확인한다.
2. 3D 화면에서 전체 적재 상태를 확인한다.
3. 3D 화면 안의 가로, 깊이, 높이 치수를 보고 선택 공간 크기를 다시 확인한다.
4. `위`, `앞`, `옆` 보기로 위치를 다시 확인한다.
5. 박스 범례를 눌러 특정 박스 유형을 강조한다.
6. 필요하면 `크게 보기`로 3D 결과를 확대한다.
7. `안전 여유 조정 추천`이 보이면 현재 적재 가능 크기와 추천 검토값을 비교한다.
8. `추천 미리보기`를 눌러 실제 설정을 바꾸기 전의 추천 3D 결과를 확인한다.
9. 결과 하단의 `현장 전달 전 점검`에서 최신 결과, `3D와 공간 확인`, 미적재/경고, 백업 파일 상태를 확인한다.

확인 포인트:

- 3D 결과가 실제로 “쌓는” 형태로 이해되는지 확인한다.
- 공중에 떠 있는 것처럼 보이는 박스가 없는지 확인한다.
- 미적재 안내가 실제 작업 판단에 도움이 되는지 확인한다.
- 안전 여유 추천은 자동 변경이 아니라 현장 책임자 확인 후 내 공간을 수정하는 안내인지 확인한다.
- 추천 미리보기에서 공간 수, 안전 여유, 3D 배치가 이해하기 쉽게 보이고, 닫아도 현재 입력이 바뀌지 않는지 확인한다.
- `현장 전달 전 점검`이 작업자가 놓치기 쉬운 3D/공간 확인과 백업 행동을 다시 안내하는지 확인한다.

### 5단계. 추가 박스 시뮬레이션

1. 결과 화면의 추가 적재 영역에서 박스를 선택한다.
2. 저장된 박스가 많으면 `저장된 박스 찾아 선택` 버튼으로 모달을 열어 필요한 박스를 고른다.
3. 각 박스의 추가 조건을 확인한다. `최대`는 가능한 만큼 계산하고, `수량 지정`은 입력한 수량까지만 계산한다.
4. 먼저 더 올려보고 싶은 박스가 있으면 `추가 우선순위`에서 `먼저 추가` 또는 `가장 먼저`를 선택한다. 같은 우선순위로 지정한 박스는 박스명 순서로 계산된다.
5. 조건을 지정하지 않았다면 `추천 결과 계산`, 수량이나 우선순위를 지정했다면 `조건 반영 결과 계산` 또는 `우선순위 결과 계산`을 눌러 `추천 결과`, `지정 우선 결과`, 박스별 우선 결과와 추가 가능 수량을 확인한다.
6. 괜찮으면 `이 결과 반영`을 누른다.

확인 포인트:

- 추가 적재 결과가 기존 적재 상태를 해치지 않는지 확인한다.
- 현장 작업자가 “추가로 몇 개 더 실을 수 있는지” 바로 이해하는지 확인한다.
- 추가 결과 미리보기 중에도 안전 여유 추천이 현재 보이는 결과 기준으로 바뀌는지 확인한다.

### 6단계. 백업 파일 만들기

1. 작업을 마친 뒤 `백업 파일 만들기`를 누른다.
2. 생성된 JSON 파일을 안전한 위치에 보관한다.
3. 다른 PC에서 이어서 확인하려면 `백업 파일 가져오기`로 해당 파일을 불러온다.

주의:

- 현재 V1은 서버 저장이 아니다.
- 같은 PC의 같은 브라우저에는 자동저장되지만, 브라우저 설정이나 저장소 정리로 사라질 수 있다.
- 현장 시연 후에는 반드시 백업 파일을 만든다.

## 8. 자주 생기는 문제

### `node` 또는 `npm` 명령어를 찾을 수 없다고 나오는 경우

원인:

- Node.js가 설치되지 않았거나, 설치 후 터미널을 새로 열지 않았다.

조치:

1. Node.js LTS를 다시 설치한다.
2. PowerShell 또는 Terminal을 모두 닫고 새로 연다.
3. `node -v`, `npm -v`를 다시 실행한다.

### `npm install`이 실패하는 경우

원인:

- 인터넷 연결 문제
- 회사 보안망 또는 프록시 문제
- npm 저장소 접속 차단
- Windows PowerShell 실행 정책이 `npm.ps1`을 차단함

조치:

1. 오류에 `npm.ps1` 또는 `이 시스템에서 스크립트를 실행할 수 없으므로`가 보이면 프로젝트 폴더에서 `cmd`를 입력한 뒤 `npm.cmd install`을 실행한다.
2. PowerShell을 계속 사용해야 한다면 `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`를 실행하고 PowerShell을 새로 연다.
3. `Get-ExecutionPolicy -List`로 `CurrentUser` 정책을 확인한다.
4. 회사 보안 정책 때문에 변경이 막히면 IT 담당자에게 Microsoft `about_Execution_Policies` 문서를 전달한다.
5. 위 오류가 아니라면 인터넷 연결을 확인한다.
6. 다른 네트워크에서 다시 시도한다.
7. 사내망이라면 IT 담당자에게 `npmjs.com` 접속 가능 여부를 확인한다.

### `http://localhost:3000`이 열리지 않는 경우

원인:

- 앱 실행 명령어가 종료됨
- 포트 충돌
- 다른 주소로 실행됨

조치:

1. 터미널에 `npm run dev`가 계속 실행 중인지 확인한다.
2. 터미널에 표시된 `Local:` 주소를 그대로 브라우저에 복사한다.
3. 포트 충돌이면 `npm run dev -- --port 3001`로 실행한다.

### `오프라인 준비`가 `준비됨`으로 바뀌지 않는 경우

원인:

- 브라우저가 서비스워커를 지원하지 않음
- 처음 앱을 열 때 인터넷 연결이 불안정함
- 회사 보안 정책 또는 브라우저 설정으로 서비스워커 등록이 차단됨

조치:

1. Chrome 또는 Microsoft Edge 최신 버전으로 다시 연다.
2. 인터넷 연결을 확인한 뒤 페이지를 새로고침한다.
3. 그래도 실패하면 앱 사용은 계속 가능하지만, 시연 종료 전 `백업 파일 만들기`를 반드시 실행한다.

### 3D 화면이 표시되지 않는 경우

원인:

- 브라우저가 WebGL을 지원하지 않거나 차단됨
- 그래픽 드라이버 문제
- 오래된 브라우저

조치:

1. Chrome 또는 Microsoft Edge 최신 버전을 사용한다.
2. 앱의 `위`, `앞`, `옆` 보기로 2D 확인을 진행한다.
3. 시연 PC의 그래픽 드라이버 또는 브라우저 설정을 확인한다.

### 백업 파일이 만들어지지 않는 경우

원인:

- 브라우저 다운로드 차단
- 다운로드 폴더 권한 또는 저장 공간 문제

조치:

1. 브라우저 다운로드 차단 안내가 보이면 허용한다.
2. 다운로드 폴더의 저장 공간을 확인한다.
3. 다시 `백업 파일 만들기`를 실행한다.

### 작업 데이터가 사라진 경우

원인:

- 브라우저 저장소 삭제
- 다른 브라우저 사용
- 다른 PC 사용
- 시크릿 모드 사용

조치:

1. 기존에 만든 백업 JSON 파일을 찾는다.
2. `백업 파일 가져오기`로 복원한다.
3. 앞으로는 시연 종료 전 반드시 `백업 파일 만들기`를 실행한다.

## 10. PM 판단

현재 상태는 현장 작업자가 제한된 시나리오로 시연해볼 수 있는 수준이다.

추천 시연 범위:

- 기본 파레트 또는 `2.5톤반` 선택
- 박스 2~3종 입력
- 결과 만들기
- 3D와 위/앞/옆 보기 확인
- 백업 파일 만들기

운영 전 현장 파일럿 확인 범위:

- 실제 현장 데이터 기준 추가 적재 정합성
- 현장 PC 기준 대량 박스 입력 시 계산 시간
- 모바일/태블릿 터치 흐름
- 브라우저 저장소 삭제 상황
- 결과 화면 문구가 실제 작업자에게 충분한지 여부

이 항목은 V1 코드 개발 잔여가 아니라 실제 현장 투입 전 파일럿에서 확인할 항목이다. V1 역사 기준은 `docs/v1-readiness.md`를 참고하되, V2 현행 결과 확인은 이 문서의 3D와 공간 확인, 미적재/경고, 백업 파일 순서를 따른다.
