# 현장형 적재 정합성 시나리오 audit

## PM 판단

V1은 현장 작업자가 실제 적재 결과를 보고 따라 할 수 있는 도구다. 따라서 UI polish보다 `공중에 떠 있는 박스`, `겹침`, `공간 경계 초과`, `깨짐주의 정책 위반`이 실제 현장형 데이터에서도 재발하지 않는다는 자동 검증 근거가 우선이다.

이번 증분은 엔진 알고리즘을 바꾸지 않고, 팔레트/20ft GP/2.5톤반 preset을 대상으로 대량 혼합 박스 시나리오를 자동 audit한다.

## 검토한 접근

1. 수동 시연 체크리스트만 운영
   - 장점: 실제 화면 감각을 확인할 수 있다.
   - 단점: 매번 누락될 수 있고 회귀를 자동으로 잡지 못한다.
2. 전체 최적화 알고리즘 재개편
   - 장점: 적재율 개선 여지가 크다.
   - 단점: V1 마무리 단계에서 회귀 위험이 크다.
3. 현장형 시나리오 자동 audit 추가
   - 장점: 기존 안전 validator를 활용해 반복 검증 가능하다.
   - 단점: 모든 실제 현장 조합을 대표하지는 않는다.

채택: 3번.

## 적용 내용

- `createFieldPackingScenarios`를 추가해 3개 현장 preset 시나리오를 만든다.
  - 파레트 기본 대량 혼합 박스
  - 20ft GP 장척 박스 혼합
  - 2.5톤반 낮은 짐칸 혼합
- `runFieldPackingScenarioAudit`가 각 시나리오를 엔진으로 계산하고 `validatePackedSpace`로 모든 공간을 검증한다.
- 미적재가 발생하거나 안전 검증에 실패한 시나리오는 `failedScenarioNames`에 기록한다.

## 역할별 검토

- product-manager: V1 종료 판단에 필요한 안전 근거를 자동화한다.
- business-analyst: 현장 기준 preset 3종을 포함해 실제 사용 맥락과 맞춘다.
- ui-designer: 화면 변경 없이 검증 근거를 보강하므로 기존 현장 UX 흐름을 흔들지 않는다.
- ui-ux-tester: 수동 시연 전 자동 audit로 공중 블록/겹침/경계 초과 회귀를 줄인다.
- code-reviewer: 새 audit은 기존 `validatePackedSpace`를 재사용해 검증 규칙 중복을 줄인다.
- nextjs-developer: 프론트 단독 V1 구조를 유지하고 순수 TypeScript 테스트로 검증한다.

## 수용 기준

- 현장 시나리오는 `파레트 기본`, `20ft GP`, `2.5톤반` preset을 모두 포함한다.
- audit은 총 3개 시나리오를 검사한다.
- 총 40개 이상의 박스를 적재 검증한다.
- 실패 시나리오는 0개여야 한다.
- `npm test`, `npx tsc --noEmit`, `npm run build`, 브라우저 검증이 통과한다.

- 총 적재 박스 수: 90
- 총 사용 공간 수: 5
- 실패 시나리오: 0
