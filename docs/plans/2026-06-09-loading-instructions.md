# 현장 적재 지시 요약

> V2 기준: 이 문서는 2026-06-09 V1 구현 이력입니다. V2에서는 배치상세/쌓는순서/작업지시서 산출물을 제거하고 3D와 공간 확인, 미적재/경고, 백업 상태 중심으로 대체했습니다.

## Goal

결과 화면의 `쌓는 순서`를 현장 작업자가 바로 따라 할 수 있는 작업 지시 문장으로 보강한다. 기존 3D/2D 뷰와 층별 요약은 유지하고, 아래층부터 어떤 박스를 놓아야 하는지 먼저 읽히게 한다.

## Context

- 핵심 사용자는 IT 도메인 지식이 거의 없는 현장 작업자다.
- 3D와 2D는 배치 검토에 유용하지만, 모바일/태블릿 현장에서는 `몇 층에 무엇을 올릴지`가 문장으로 바로 보여야 한다.
- WCAG 2.2의 포인터 타깃 최소 기준은 24 CSS px이지만, 이 제품은 현장 터치 환경을 고려해 기존 44~48px 기준을 유지한다.
- 참고: W3C Understanding SC 2.5.8 Target Size Minimum, https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum

## 3 Approaches

1. **기존 층별 요약만 유지**
   - 장점: 구현 변경이 없다.
   - 단점: 작업자가 "그래서 실제로 무엇부터 놓지?"를 직접 해석해야 한다.
   - 판단: 현장 UX 개선 요구에 부족하다.

2. **별도 작업 지시 페이지 추가**
   - 장점: 인쇄/공유/작업 지시서 확장성이 높다.
   - 단점: V1 프론트 단독 현재 화면 흐름에서는 새 화면과 상태 전달 범위가 커진다.
   - 판단: 후속 증분으로 적합하다.

3. **결과 하단에 현장 작업 순서 문장 추가**
   - 장점: 기존 결과 데이터와 층 요약을 재사용하며, 모바일에서 바로 읽힌다.
   - 단점: 결과 하단 정보량이 조금 늘어난다.
   - 판단: 이번 증분 채택안이다.

## Role Review

- product-manager: 결과 신뢰성과 현장 실행성을 높이는 작은 증분이다. 엔진 좌표나 저장 스키마를 바꾸지 않아 리스크가 낮다.
- business-analyst: 작업자가 `바닥에 먼저 놓기`, `Nmm 높이에 올리기`처럼 행동 단위로 이해할 수 있다.
- ui-designer: 기존 `쌓는 순서` 패널 안에 작업 지시를 먼저 두고, 층별 요약은 보조 목록으로 유지한다.
- ui-ux-tester: 모바일에서는 한 컬럼으로 접고 긴 박스명은 줄바꿈되어야 한다. 상호작용 버튼을 추가하지 않으므로 터치 타깃 회귀 위험은 낮다.
- nextjs-developer: `stacking-layer-summary` 순수 유틸을 확장하고, 결과 컴포넌트는 memoized view model만 렌더링한다.

## Acceptance Criteria

- 선택 공간의 적재 박스가 있으면 결과 하단에 `현장 작업 순서` 목록이 표시된다.
- 바닥층은 `바닥에 먼저 놓습니다`, 위층은 `Nmm 높이에 올립니다` 문장으로 표시된다.
- 같은 층에 유형이 많으면 기존 요약 규칙처럼 대표 유형과 `외 N종`으로 압축한다.
- 모바일 360px/390px에서 지시 문장과 긴 박스명이 가로 스크롤 없이 줄바꿈된다.
- 기존 3D/2D 뷰, 공간 선택, 박스 범례, 체이닝 흐름은 변경하지 않는다.
