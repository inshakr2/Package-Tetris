# 안전 여유 조정 추천

## Product Manager Decision

사용자 아이디어는 제품 목적과 잘 맞는다. 공간 하나가 아쉽게 더 필요한 경계 상황에서는 안전 여유를 일부 조정해도 되는지 확인하는 것만으로 차량/파레트 수를 줄일 수 있다.

검토한 방법:

1. 결과 생성 시 offset 후보를 자동 시뮬레이션하고 추천 카드 표시
   - 장점: 사용자가 별도 조작 없이 개선 가능성을 알 수 있다.
   - 단점: 안전 여유를 자동으로 줄였다고 오해하지 않도록 문구가 필요하다.
2. 결과 화면에 offset 수동 조정 패널 추가
   - 장점: 사용자가 직접 여러 값을 시험할 수 있다.
   - 단점: V1 화면 복잡도와 오입력 위험이 크다.
3. 추가 적재 시뮬레이션마다 별도 offset 슬라이더 제공
   - 장점: 최적화 탐색 범위가 넓다.
   - 단점: 계산 비용과 사용성 부담이 V1에는 크다.

채택: 1번. V1에서는 자동 적용이 아니라 추천 카드로 시작한다. 후보는 10/20/50mm 안전 여유 완화만 검사하고, 미적재 없이 공간 수가 줄어드는 경우에만 보여준다.

추가 안정화 결정: 추천 카드에는 `공간 설정 확인` 아래 `추천 미리보기`를 제공한다. 이 버튼은 실제 공간 설정을 바꾸지 않고, 추천 offset으로 다시 계산된 결과를 dialog 안에서 3D로 확인하게 한다.

## Role Review

- code-reviewer: offset은 안전 여유이므로 자동 변경하면 안 된다. 추천 결과도 실제 적재 결과와 같은 안전 게이트를 통과해야 한다.
- business-analyst: 현장에서는 공간 수 절감이 비용과 직결된다. 단, 책임자가 안전 여유 조정을 승인해야 한다는 문구가 필요하다.
- ui-ux-tester: 추천은 결과 요약 근처의 한 카드로 노출하고, 모바일에서 긴 수치가 가로 넘침을 만들지 않아야 한다.
- ui-ux-tester: 미리보기 dialog는 실제 설정이 바뀌지 않았다는 문구를 상단에 두고, 모바일에서는 공간 선택과 3D 확인이 한 컬럼으로 이어져야 한다.
- nextjs-developer: 추천 계산은 기존 worker 기반 적재 계산을 재사용한다. ResultSummary schema는 바꾸지 않고 화면 상태로만 관리한다.

## Acceptance Criteria

- 결과가 2개 이상 공간을 쓰고, 안전 여유를 줄인 후보가 더 적은 공간을 쓰면 추천 카드를 표시한다.
- 후보는 10mm, 20mm, 50mm 순서로만 검사한다.
- 후보 결과에 미적재가 생기면 추천하지 않는다.
- 추천 카드는 현재 적재 가능 크기와 추천 검토값을 함께 보여준다.
- 추천 카드는 `현장 책임자 확인 후`라는 안전 문구를 포함한다.
- 추가 결과 미리보기 중에는 미리보기 공간 기준으로 같은 추천 검사를 수행한다.
- `추천 미리보기`는 추천 offset으로 계산된 공간 목록을 보여주되, 저장된 공간이나 현재 입력을 변경하지 않는다.
- 추천 후보는 개선 후보를 찾으면 이후 후보 계산을 중단해 불필요한 계산을 줄인다.
