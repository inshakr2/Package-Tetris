# 결과 뷰어 모바일 전환 버튼 개선

## PM 판단

- 현장 작업자는 결과 확인 중 `3D`, `위`, `앞`, `옆`, `처음` 전환을 반복해서 누를 가능성이 높다.
- 현재 버튼은 flex wrap 기반이라 좁은 모바일에서 줄바꿈 위치가 화면 폭에 따라 흔들릴 수 있다.
- 2026-06-09 후속 검증에서 360px 폭의 일부 3D 카메라 버튼이 내부 폭을 넘는 위험이 확인되어, 가로 chip row 대신 2열 고정 그리드로 조정했다.
- 이번 증분은 풀스크린 3D나 제스처 안내까지 확장하지 않고, 결과 뷰어 전환 버튼의 터치 안정성만 개선한다.

## 적용 방향

- 결과 보기 버튼은 모바일에서 2열 그리드로 고정한다.
- 3D 카메라 버튼은 3D 보기 상태에서만 같은 2열 그리드로 표시한다.
- `크게 보기` 버튼은 2열 중 한 칸에 끼우지 않고 전체 행을 사용해 360px에서도 잘리지 않게 한다.
- 버튼 라벨은 현장 언어로 짧게 유지한다: `3D`, `위`, `앞`, `옆`, `처음`, `사시`.
- 선택 상태는 기존 `aria-pressed`, 테두리, 배경, 굵은 글자 표현을 유지한다.
- 터치 타깃은 모바일에서 최소 48px 높이를 유지한다.

## 역할별 검토 반영

- business-analyst: 모바일/태블릿 결과 영역에서 버튼 겹침과 보기 전환 후 선택 상태 유지를 수용 기준으로 둔다.
- ui-designer: 아이콘만 사용하는 방식은 피하고 짧은 텍스트 라벨을 유지한다. `처음`은 마지막에 둔다.
- ui-ux-tester: 360px, 390px, 768px, 1280px 폭에서 overflow, 터치 타깃, 3D 캔버스 유지 여부를 확인한다.

## 개발 수용 기준

- `RESULT_VIEW_CONTROL_ITEMS`와 `THREE_CAMERA_CONTROL_ITEMS`는 테스트로 순서와 라벨을 고정한다.
- `reset` 액션은 projection view와 타입 수준에서 분리한다.
- 767px 이하에서 결과 보기 버튼과 3D 카메라 버튼은 2열 그리드와 48px 이상 가로/세로 터치 타깃을 유지한다.
- 브라우저 검증에서 모바일 폭의 페이지 가로 overflow가 없어야 한다.
