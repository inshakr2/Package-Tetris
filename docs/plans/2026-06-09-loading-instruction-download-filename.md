# Loading Instruction Download Filename Status

> V2 기준: 이 문서는 2026-06-09 V1 구현 이력입니다. V2에서는 배치상세/쌓는순서/작업지시서 산출물을 제거하고 3D와 공간 확인, 미적재/경고, 백업 상태 중심으로 대체했습니다.

## Goal

`작업 지시서 저장`을 누른 뒤 현장 작업자가 다운로드 폴더에서 어떤 파일을 찾아야 하는지 바로 알 수 있게 한다.

## Product Review

- product-manager: 클립보드가 막힌 환경을 대비해 텍스트 파일 저장을 추가했지만, 저장 후 파일명이 보이지 않으면 현장 사용자가 파일을 찾는 데 실패할 수 있다.
- business-analyst: 사용자는 브라우저 다운로드 위치나 파일명 규칙을 모를 수 있으므로, 완료 문구는 다음 행동까지 말해야 한다.
- ui-designer: 별도 모달이나 큰 안내를 띄우지 않고 기존 상태 문구를 확장하는 것이 결과 화면 흐름을 덜 방해한다.
- ui-ux-tester: 긴 파일명은 모바일에서 줄바꿈되어야 하고, 버튼 높이와 결과 패널 레이아웃을 깨지 않아야 한다.
- nextjs-developer: 파일명 생성 로직은 컴포넌트 내부 함수보다 순수 유틸로 분리해 날짜/공간 index를 테스트한다.

## Compared Approaches

1. 현재처럼 `파일을 만들었습니다`만 표시한다.
   - 장점: 변경 없음.
   - 단점: 현장 사용자가 다운로드 폴더에서 파일을 찾기 어렵다.
2. 완료 상태에 파일명과 다운로드 폴더 확인 문구를 표시한다.
   - 장점: 가장 작은 변경으로 다음 행동을 명확히 한다.
   - 단점: 상태 문구가 조금 길어진다.
3. 저장 완료 후 별도 dialog를 띄운다.
   - 장점: 눈에 잘 띈다.
   - 단점: 결과 확인 흐름을 끊고 모바일 조작 단계가 늘어난다.

선택: 2번. 기존 상태 영역을 재사용해 `다운로드 폴더에서 <파일명> 파일을 확인하세요.`를 표시한다.

## Acceptance Criteria

- 파일명 생성은 `space index`와 로컬 날짜를 사용한다.
- 선택 공간이 없으면 `space` 기본 라벨을 사용한다.
- 저장 성공 문구는 다운로드 폴더와 실제 파일명을 함께 알려준다.
- 모바일에서 완료 문구는 줄바꿈되어 패널 밖으로 넘치지 않는다.
- 전체 테스트, 타입체크, 빌드, 브라우저 검증을 통과한 뒤에만 커밋/푸시한다.
