# Loading Instruction Warning Copy Plan

> V2 기준: 이 문서는 2026-06-09 V1 구현 이력입니다. V2에서는 배치상세/쌓는순서/작업지시서 산출물을 제거하고 3D와 공간 확인, 미적재/경고, 백업 상태 중심으로 대체했습니다.

## Context

작업자가 `작업 순서 복사`로 받은 문장만 보고 적재를 진행하면 화면에 있던 미적재/안전 경고를 놓칠 수 있다. 이번 증분은 복사 텍스트 안에 `확인 필요` 문장을 함께 포함해, 화면 밖으로 전달된 지시서도 안전 맥락을 유지하도록 한다.

참고 자료:
- W3C WCAG Labels or Instructions: 사용자가 입력/작업을 이해할 수 있도록 필요한 안내를 제공해야 한다.
- MDN ARIA status role: 상태 문구는 사용자의 흐름을 끊지 않는 advisory information으로 전달할 수 있다.

## Compared Approaches

1. **복사 텍스트에 경고 줄 포함**
   - 장점: 기존 버튼과 워크플로우를 그대로 유지하면서 전달 누락 위험을 줄인다.
   - 단점: 복사 문장이 길어질 수 있어 최대 표시 건수 제한이 필요하다.

2. **경고가 있으면 복사 차단**
   - 장점: 위험 상황에서 작업 지시 전달을 강하게 막을 수 있다.
   - 단점: 현장에서는 미적재를 알고 부분 적재만 진행해야 하는 경우도 있어 과도하게 막힐 수 있다.

3. **별도 경고 확인 모달**
   - 장점: 사용자가 경고를 더 명확히 인지한다.
   - 단점: 작은 증분 대비 조작 단계가 늘고 모바일 흐름이 무거워진다.

선택: 1번. 복사 텍스트에 미적재 수량과 주요 경고를 `확인 필요` 줄로 넣고, 경고가 많으면 `외 N건`으로 압축한다.

## Role Review

- product-manager: 복사 기능의 실제 현장 위험을 줄이는 바로 다음 증분으로 적합하다.
- business-analyst: 화면을 보지 않는 작업자에게도 미적재와 안전 경고가 전달되어야 한다.
- ui-designer: UI 버튼은 추가하지 않고 복사 결과만 보강해 화면 복잡도를 늘리지 않는다.
- ui-ux-tester: 경고가 있어도 버튼 상태와 성공/실패 상태 문구가 기존처럼 동작해야 한다.
- nextjs-developer: `createStackingInstructionText` 옵션만 확장하고 컴포넌트는 기존 경고 요약을 전달한다.
- code-reviewer: 중복 경고, 과도한 텍스트 길이, 빈 결과 복사, 기존 테스트 회귀를 확인한다.

## Acceptance Criteria

- 미적재가 있으면 복사 텍스트 상단에 미적재 수량 확인 문장이 포함된다.
- 안전 경고가 있으면 복사 텍스트 상단에 `확인 필요` 문장으로 포함된다.
- 중복 경고는 한 번만 포함된다.
- 경고가 많으면 최대 표시 건수 뒤 `외 N건`으로 압축된다.
- UI 레이아웃과 3D 결과 검증은 기존과 동일하게 유지된다.
