# 결과 상세 확인 모달 개선

## PM 판단

- 현장 작업자는 공간을 바꿔가며 3D 적재 결과, 배치 상세, 쌓는 순서를 반복 확인한다.
- 하단 고정 패널 방식은 공간 선택 버튼과 상세 정보 사이의 스크롤 왕복을 만든다.
- V1 프론트 단독 범위에서는 선택 공간 상태를 그대로 공유하는 모달 방식이 가장 작은 변경으로 사용성을 개선한다.

## 검토한 방법

1. 하단 상세 영역 근처에 공간 전환 버튼을 추가한다.
   - 장점: 기존 패널 구조를 유지할 수 있다.
   - 단점: 상단 공간 선택과 동기화해야 해서 같은 제어가 중복된다.
2. 3D 보기 근처에 `배치 상세`, `쌓는 순서` 버튼을 두고 모달로 확인한다.
   - 장점: 사용자가 결과를 보던 위치에서 상세 내용을 바로 확인한다.
   - 단점: 모달의 포커스 복귀와 모바일 전체 화면 처리가 필요하다.
3. 결과 영역 안에 탭을 추가한다.
   - 장점: 한 화면 안에서 상세와 뷰어를 전환할 수 있다.
   - 단점: 3D 자체가 메인인 V1 화면에서 뷰어 면적을 줄일 위험이 있다.

## 적용 방향

- 방법 2를 적용한다.
- 3D 보기와 2D 보기 컨트롤 아래에 `배치 상세`, `쌓는 순서` 버튼을 둔다.
- 버튼은 현재 선택된 공간 결과를 기준으로 열리며, 상단 공간 선택과 별도 동기화 버튼을 만들지 않는다.
- 모달을 닫으면 열었던 버튼으로 포커스를 되돌린다.
- 모바일에서는 모달을 전체 화면에 가깝게 열어 표와 작업 순서가 좁은 폭에서 잘리지 않게 한다.
- 하단의 기존 `배치 상세`, `쌓는 순서` 패널은 제거해 결과 화면의 세로 길이를 줄인다.

## 역할별 검토

- business-analyst: 현장 사용자는 결과 확인 중 다른 공간을 반복 선택하므로 스크롤 왕복을 줄이는 방식이 우선이다.
- ui-designer: 상세 확인 버튼은 결과 뷰어 가까이에 두고, 아이콘과 텍스트를 함께 사용해 의미를 바로 알 수 있게 한다.
- ui-ux-tester: 360px, 390px, 768px, 1280px에서 버튼 터치 타깃, 모달 표시, 가로 overflow, 닫기 후 포커스 복귀를 확인한다.
- code-reviewer: 모달은 현재 선택 공간 데이터만 읽고, 적재 알고리즘과 IndexedDB 저장 흐름을 변경하지 않는다.

## 개발 수용 기준

- 3D 카메라 첫 프리셋 라벨은 `자유시점`으로 표시한다.
- 키보드 안내와 문서에도 `자유시점`을 사용한다.
- `배치 상세`, `쌓는 순서`는 결과 뷰어 근처 버튼으로 열리는 dialog 안에서 표시한다.
- 기존 하단 상세 패널은 결과 화면에서 제거한다.
- dialog 안의 표와 순서 목록은 모바일에서 가로 스크롤 없이 읽을 수 있어야 한다.
