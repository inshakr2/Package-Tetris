# Storage Panel Field Copy Plan

## 목적

저장 보호 패널 안의 기술 용어를 현장 작업자가 이해할 수 있는 말로 바꾸고, 숨겨져 있던 저장 상태 세부 안내를 실제 화면에 보여준다.

## 검토한 3가지 접근안

1. 기존 문구 유지
   - 장점: 구현 변경이 없다.
   - 단점: `revision` 같은 개발자 용어가 남고, 충돌 배너와 패널 문구가 서로 달라진다.

2. 컴포넌트 내부 문자열만 직접 수정
   - 장점: 빠르게 바꿀 수 있다.
   - 단점: 저장 상태별 copy를 테스트하기 어렵고, 같은 작업본 번호 표현이 다시 어긋날 수 있다.

3. 저장 상태 copy를 순수 helper로 분리하고 패널 detail을 렌더링한다.
   - 장점: 충돌, 저장 중, 다른 탭 감지 상태를 테스트로 고정할 수 있다.
   - 장점: 충돌 배너와 같은 `작업본 번호` 표현을 재사용할 수 있다.
   - 단점: 작은 helper 파일이 추가된다.

## 채택안

3번을 채택한다. `createLocalSaveState` helper가 저장 상태 row의 tone, value, description, detail을 계산하고, UI는 이 결과를 그대로 표시한다.

## 수용조건

- 저장 충돌 detail은 `작업본 번호: 저장된 최신 N · 현재 화면 N`으로 표시된다.
- 저장 보호 패널 본문에는 `revision`, `저장소 revision`, `저장 기준 revision` 같은 개발자 용어가 직접 노출되지 않는다.
- 다른 탭이 열린 저장 중 상태에서는 오래된 내용 덮어쓰기 방지 안내가 표시된다.
- 저장 상태 row의 detail은 `StorageHealthRow`에 실제로 전달된다.
- 모바일 360px/390px에서 저장 보호 패널이 가로 overflow를 만들지 않는다.
