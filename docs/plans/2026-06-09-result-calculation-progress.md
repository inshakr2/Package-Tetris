# Result Calculation Progress Plan

## Product-Manager Decision

Package Tetris V1은 현장 작업자가 버튼을 누른 뒤 화면이 멈춘 것처럼 느끼지 않도록 결과 생성 상태를 단계별로 보여준다. 서버 작업이나 장기 작업 큐는 V1 범위 밖이므로 프론트 상태만으로 `검증 중 -> 적재 계산 중 -> 3D 생성 중`을 표시한다.

## Reviewed Approaches

1. 버튼 문구만 `결과 계산 중...`으로 유지한다.
   - 구현은 가장 작지만 계산 대기 중 어떤 일을 하는지 알 수 없다.
2. 화면 전체 로딩 오버레이를 띄운다.
   - 눈에 잘 띄지만 현장 사용자가 기존 입력과 결과 맥락을 보지 못한다.
3. 실행 전 검토와 결과 영역에 작은 status 안내를 추가한다.
   - 기존 흐름을 유지하면서 현재 단계를 설명할 수 있어 V1에 적합하다.

선택: 3번. CTA 문구와 status 안내를 같은 유틸에서 가져와 문구 불일치를 줄이고, 계산 흐름은 짧은 tick으로 나누어 실제 화면에 단계 변화가 반영될 수 있게 한다.

## Role Review

- Business Analyst: 사용자는 계산이 오래 걸릴 때 중복 클릭보다 현재 상태 확인을 원한다. `입력 조건`, `빈 공간`, `3D 보기`처럼 현장 행동과 연결되는 문구가 필요하다.
- UI Designer: 오버레이 대신 기존 카드 안에 compact status를 둔다. 모바일 폭을 밀지 않도록 한 줄 고정 문구를 피하고 줄바꿈을 허용한다.
- UI/UX Tester: status 영역은 `role="status"`와 `aria-live="polite"`로 제공한다. CTA는 계산 중 비활성화되고 같은 단계 문구를 보여야 한다.
- Code Reviewer: 순수 copy 유틸 테스트와 컴포넌트 연결 테스트를 추가한다. 계산 완료, 실패, 저장 충돌 경로에서 진행 상태가 `idle`로 복귀해야 한다.

## Acceptance

- 결과 생성 중 `검증 중`, `적재 계산 중`, `3D 생성 중` 단계 문구를 제공한다.
- 실행 전 검토 카드와 결과 영역의 계산 CTA는 같은 진행 문구를 사용한다.
- status 안내는 모바일에서도 레이아웃을 밀지 않고, 화면 읽기 환경에 polite live region으로 노출된다.
- 기존 IndexedDB 저장, 결과 생성, 3D 결과 확인 흐름을 변경하지 않는다.
