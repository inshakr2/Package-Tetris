# 현재 작업 새로 시작

## Product Manager Decision

현장 시연 중 같은 브라우저에 이전 작업 박스와 결과가 남아 있으면 작업자가 `저장된 박스` 라이브러리와 `이번 작업`을 혼동할 수 있다. V1은 서버 없이 IndexedDB에 데이터를 유지하므로, 저장 라이브러리를 지우지 않으면서 현재 작업만 새로 시작하는 경로가 필요하다.

검토한 방법:

1. 브라우저 저장소 전체 초기화
   - 내 공간과 저장된 박스까지 삭제되어 현장 데이터 손실 위험이 크다.
2. 현재 작업만 초기화
   - 저장된 공간/박스는 유지하고 선택 공간, 이번 작업 박스, 결과, 추가 적재 기록만 비운다.
3. 새 파일 ID로 복사본 시작
   - 추후 동기화 모델과는 맞지만 V1 화면에서는 파일 개념이 무겁다.

채택: 2번. `새 작업 시작`은 라이브러리를 보존하는 현재 작업 초기화로 구현한다.

## Role Review

- business-analyst: 현장 사용자는 저장한 박스와 이번 작업 박스를 구분해야 한다. `저장된 공간과 박스는 그대로 둡니다.` 문구가 핵심이다.
- ui-ux-tester: destructive 성격이 있으므로 즉시 실행하지 않고 확인 dialog를 사용한다. 모바일에서는 버튼과 dialog 액션이 한 컬럼으로 내려가야 한다.
- code-reviewer: reset 로직은 순수 유틸로 분리하고, `spaces`, `blockTemplates`, `policy`, `fileId`, `deviceId` 보존을 테스트로 고정한다.
- nextjs-developer: V1 프론트 단독 범위에서 IndexedDB 저장 루프를 그대로 사용한다. 서버나 새 저장소 스키마는 추가하지 않는다.

## Acceptance Criteria

- 현재 작업 영역에 `새 작업 시작` 액션이 보인다.
- 작업 박스, 결과, 추가 적재 기록, 기본이 아닌 선택 공간이 있으면 액션이 활성화된다.
- 확인 dialog는 현재 작업 박스, 계산 결과, 추가 적재 기록만 비우고 저장된 공간/박스는 유지한다고 안내한다.
- 확인 후 draft는 기본 파레트, 빈 작업 박스, `space` 단계로 돌아간다.
- 최근 결과와 체이닝 이력은 비워진다.
- 저장된 공간/박스 라이브러리와 정책은 유지된다.
- 모바일에서 가로 넘침 없이 48px 터치 타깃을 유지한다.
