# Chain Result Validation Plan

**Goal:** 추가 적재 시뮬레이션이 과거 백업이나 이전 엔진 결과의 잘못된 좌표를 그대로 신뢰하지 않도록 한다.

## Product Manager Decision

채택 접근안은 `체이닝 시작 전 기존 결과 정합성 검증`이다.

1. 기존 결과를 그대로 신뢰
   - 과거 결과에 공중 블록이나 겹침이 있으면 추가 적재가 잘못된 기준 위에서 계속 계산된다.
2. 체이닝 중 새 블록만 검증
   - 새 블록은 안전해도 기준 결과가 잘못되어 있으면 현장 작업자는 전체 결과를 신뢰할 수 없다.
3. 기존 결과 전체 검증
   - 추가 적재 전에 기준 결과를 먼저 검사하고, 안전 기준을 만족하지 않으면 계산을 중단한다.

## Scope

- 기존 결과의 모든 공간을 usable size 기준으로 검증한다.
- 같은 공간 안의 박스끼리 3D overlap이 있으면 invalid 처리한다.
- `zMm > 0` 블록은 하부 박스 상단면이 바닥 전체를 받쳐야 한다.
- 일반 박스는 깨짐주의 박스 위에 올라갈 수 없다.
- 깨짐주의끼리 쌓기 가능 여부는 현재 작업 policy를 따른다.
- invalid 결과에서는 추가 적재를 계산하지 않고 기존 spaces를 그대로 반환한다.
- UI는 invalid warning을 `더 들어가지 않습니다`보다 우선 표시한다.
- UI는 invalid warning을 preview로 반영하지 않고, 체이닝 패널에 `결과 다시 생성` 액션을 제공한다.

## Acceptance Criteria

- 공중에 떠 있는 기존 결과는 체이닝이 거부된다.
- 겹치는 기존 결과는 체이닝이 거부된다.
- 깨짐주의 정책을 어긴 기존 결과는 체이닝이 거부된다.
- 정상 결과는 기존 체이닝 시뮬레이션을 계속 수행한다.
- 작업자는 `기존 결과의 배치가 안전 기준에 맞지 않아 추가 적재를 계산할 수 없습니다. 결과를 다시 생성하세요.` 문구로 다음 행동을 이해할 수 있다.
- 검증 실패 상태에서 `최대 적재 계산` 반복 대신 `결과 다시 생성`으로 이동할 수 있다.
- `npm test`, `npx tsc --noEmit`, `npm run build`가 통과한다.
