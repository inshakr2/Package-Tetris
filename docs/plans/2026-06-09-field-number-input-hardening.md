# Package Tetris V1 Field Number Input Hardening

## Goal

현장 작업자가 모바일/태블릿에서 공간 크기, 안전 여유, 박스 크기, 수량을 입력할 때 빈 입력이나 잘못된 숫자가 작업본에 섞이지 않도록 막는다.

## Options Reviewed

1. **숫자 입력 전용 React 컴포넌트로 통합**
   - 장점: 라벨, 단위, 입력 보정, 접근성을 한 곳에서 관리할 수 있다.
   - 단점: 현재 V1 마무리 시점에는 컴포넌트 교체 범위가 커지고 회귀 검증 범위도 넓어진다.

2. **폼 전체 상태를 문자열로 전환**
   - 장점: 사용자가 입력을 지우는 중간 상태를 가장 자연스럽게 표현한다.
   - 단점: 저장, 검토, 계산 입력 변환까지 연쇄 변경이 필요하다.

3. **필드 내부 문자열 상태 + 계산용 숫자 상태 유지**
   - 장점: 기존 구조를 유지하면서 빈 입력, 문자, 소수, 비정상 가져오기 데이터를 막을 수 있다.
   - 단점: 숫자 입력 컴포넌트가 별도 편집 상태를 관리하므로 테스트로 고정해야 한다.

## Decision

3번을 적용한다. V1은 현장 시연 가능한 안정성이 우선이므로 작업본의 계산용 상태는 숫자로 유지하되, 각 숫자 필드는 내부 문자열 편집 상태를 가진다. 사용자는 값을 지우고 다시 입력할 수 있으며, 빈 값/소수/문자는 필드 아래 오류로 즉시 안내한다. 유효한 정수만 작업본에 반영하고, 저장 파일 가져오기처럼 비신뢰 데이터가 들어오면 실행 전 검토에서 차단한다.

## Role Review

- **product-manager:** 프론트 단독 V1 범위 안에서 시연 리스크를 줄이는 증분이다. 서버나 저장 구조 변경은 필요 없다.
- **business-analyst:** IT 지식이 없는 현장 사용자는 단위와 입력 오류 원인을 즉시 이해해야 한다. `mm`, `개`를 화면과 접근성 라벨에 명시한다.
- **ui-designer:** 화면 밀도는 유지하되 수량 라벨을 `기본 수량(개)`, `이번 작업 수량(개)`처럼 명확하게 바꾼다.
- **ui-ux-tester:** 모바일 숫자 키패드, clear-then-type 값 교체, 소수/문자 오류 표시, 1 단위 스텝, 48px 터치 타깃을 확인한다.
- **code-reviewer:** 저장 파일 가져오기처럼 비신뢰 데이터가 들어와도 `NaN`, `Infinity`, 0 미만 값이 계산 입력이나 화면 부피 표시로 넘어가지 않아야 한다.

## Acceptance

- 숫자 입력을 지운 뒤 새 숫자를 입력하면 기존 값 뒤에 붙지 않고 새 값으로 교체된다.
- 빈 값, 소수, 문자는 작업본에 반영하지 않고 필드 아래에 `숫자를 입력하세요.` 또는 `정수만 입력하세요.`를 표시한다.
- 수량은 `개`, 치수와 안전 여유는 `mm` 단위를 화면 라벨과 접근성 라벨에 포함한다.
- 복원된 작업본에 `NaN` 또는 `Infinity`가 있어도 실행 전 검토는 차단하고, 현재 작업 카드의 부피 표시에는 `NaNm³`/`Infinitym³`가 노출되지 않는다.
- 모바일 360px/390px, 태블릿 768px, 데스크톱 1280px에서 가로 넘침이 없고 결과 3D canvas가 렌더링된다.

## Verification

- `npm test`
- `npx tsc --noEmit`
- `npm run build`
- Browser: 360px, 390px, 768px, 1280px에서 숫자 입력 라벨, overflow, 결과 3D 렌더링 확인
