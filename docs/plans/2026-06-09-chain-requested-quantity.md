# 추가 박스 시뮬레이션 수량 직접 지정

## PM 판단

V1 기획서는 추가 박스 시뮬레이션에서 `최대 적재 계산`을 기본 액션으로 두되, 현장 작업자가 원하는 수량만 시험할 수 있는 보조 액션도 제공한다고 정리한다. 현재 화면은 최대 적재만 지원해 `3개만 더 넣으면 가능한가?` 같은 현장 질문을 바로 확인하기 어렵다.

이번 증분은 새 공간 생성이나 수동 배치 편집 없이, 기존 결과 공간 안에서 선택한 박스를 요청 수량까지만 추가 배치한다.

V2에서는 현장 피드백에 따라 단일 박스 수량 입력을 확장했다. 선택한 최대 3개 박스 각각에 대해 `최대` 또는 `수량 지정` 조건을 설정하고, `추천 결과 계산` 한 번으로 추천 결과 / 각 박스 우선 결과를 비교한다. 예를 들어 A는 10개까지만, B와 C는 최대 수량으로 계산할 수 있다.

## 검토한 접근

1. 최대 적재 계산만 유지하고 결과 문구를 보강한다.
   - 장점: 구현 변경이 없다.
   - 단점: 원하는 수량을 시험하려는 현장 질문을 해결하지 못한다.
2. 수량 입력과 `지정 수량 계산` 버튼을 추가한다.
   - 장점: 기존 엔진 흐름을 재사용하면서 원하는 수량 확인을 바로 지원한다.
   - 단점: 다중 박스 조합에서는 어떤 박스에 대한 수량인지 불명확해진다.
3. 여러 수량 결과를 한 번에 비교하는 UI를 만든다.
   - 장점: 분석 기능은 강하다.
   - 단점: V1 현장 사용성 대비 화면과 검증 범위가 커진다.
4. 선택 박스별로 `최대`/`수량 지정` 조건을 제공하고 추천 계산에 함께 반영한다.
   - 장점: A는 고정 수량, B/C는 최대 수량 같은 현장 조합을 한 번에 계산할 수 있다.
   - 단점: UI와 엔진 입력 계약이 함께 바뀐다.

채택: V1 당시에는 2번을 적용했으나, V2 현장 피드백 이후 4번으로 확장한다.

## 적용 범위

- `runChainSimulationV0`에 `requestedQuantity`를 선택 입력으로 유지한다.
- `runMultiChainSimulationV0`에는 `requestedQuantitiesByTemplateId`를 추가해 박스별 지정 수량을 적용한다.
- 값이 없는 박스는 기존처럼 최대 적재량까지 계산한다.
- 값이 있는 박스는 최대 적재 가능량과 요청 수량 중 작은 값까지만 추가한다.
- 요청 수량이 1개 미만이면 계산하지 않고 현장 문구로 안내한다.
- 체이닝 패널에는 선택 박스별 `최대`/`수량 지정` 컨트롤을 제공한다.

## 역할별 검토

- business-analyst: 현장 작업자는 최대치보다 실제 남은 주문 수량이나 추가 출고 수량을 확인하는 경우가 많다.
- ui-designer: 선택 박스별 조건을 작은 행 단위로 묶고, 태블릿 이하에서는 한 컬럼으로 접어 가독성을 유지한다.
- ui-ux-tester: 모바일 360px/390px에서 최대/수량 지정 버튼과 입력이 48px 터치 타깃을 유지해야 한다.
- code-reviewer: 최대 적재 기존 동작은 그대로 유지하고, 요청 수량 제한은 순수 엔진 테스트로 고정한다.
- nextjs-developer: 서버, 저장 스키마, 체이닝 이력 구조는 바꾸지 않는다.

## 수용 기준

- 최대 적재 계산은 기존 결과를 유지한다.
- 박스별 지정 수량은 해당 박스의 요청 수량 이하만 추가한다.
- 지정하지 않은 박스는 최대 수량으로 계산한다.
- 요청 수량 0 이하 입력은 추가 계산을 하지 않는다.
- 모바일/태블릿에서 수량 입력과 버튼이 가로 넘침 없이 표시된다.
- `npm test`, `npx tsc --noEmit`, `npm run build`, 브라우저 검증을 통과한 뒤 커밋/푸시한다.
