# Base Area Priority Packing Plan

**Goal:** 같은 안전 규칙 안에서 넓은 받침면이 될 수 있는 박스를 먼저 배치해 불필요한 공간 분리를 줄인다.

## Product Manager Decision

채택 접근안은 `취급 그룹 내 최대 바닥면적 우선 정렬`이다.

1. 기존 부피 우선 유지
   - 좁고 높은 박스가 낮고 넓은 박스보다 먼저 배치될 수 있어, 넓은 박스가 바닥 또는 충분한 받침면을 잃을 수 있다.
2. 전역 최적화 탐색
   - 더 좋은 결과를 낼 수 있지만 구현 복잡도와 계산 시간이 커지고, 현장용 V1 안정성 검증 범위를 넓힌다.
3. 최대 바닥면적 우선 휴리스틱
   - 현재 결정론적 엔진을 유지하면서, 받침면이 될 가능성이 큰 박스를 먼저 깔아 후속 박스가 안정적으로 올라갈 가능성을 높인다.

## Scope

- non-fragile 우선, fragile 후순위 정책은 유지한다.
- 같은 fragile 그룹 안에서 회전 가능한 후보 중 최대 바닥면적이 큰 박스를 먼저 배치한다.
- 최대 바닥면적이 같으면 부피가 큰 박스를 먼저 배치한다.
- 부피도 같으면 `blockId`로 정렬해 결과 결정론을 유지한다.
- full support, 3D overlap, usable size, 깨짐주의 정책은 완화하지 않는다.

## Acceptance Criteria

- `1000 / 1000 / 150mm` 판형 박스와 `500 / 500 / 700mm` 고형 박스는 한 공간에 배치된다.
- 판형 박스는 바닥에 먼저 놓이고, 고형 박스는 판형 박스 위에 안정적으로 올라간다.
- 기존 안정성 테스트는 모두 계속 통과한다.
- `npm test`, `npx tsc --noEmit`, `npm run build`가 통과한다.
