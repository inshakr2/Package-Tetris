# 현장 시연 예제 불러오기

## Product Manager Decision

V1은 현장 작업자가 실제로 시연해볼 수 있어야 한다. 하지만 첫 시연에서 공간과 여러 박스를 모두 손으로 입력해야 하면 결과 화면과 3D 검토까지 도달하기 전에 시간이 오래 걸린다.

검토한 방법:

1. 문서에 예제 입력값만 추가
   - 구현 리스크는 없지만 비개발자 시연자가 여전히 모든 값을 직접 입력해야 한다.
2. 현재 작업 영역에 `시연 예제 불러오기` 버튼 추가
   - 한 번에 2.5톤반과 예제 박스를 채우고 바로 `결과 만들기`로 이어질 수 있다.
   - 저장된 내 공간과 내 박스는 보존하되 현재 작업, 결과, 추가 적재 기록만 시연 데이터로 교체한다.
3. 별도 데모 모드 라우트 추가
   - 장기적으로는 유용하지만 V1 단일 화면 흐름과 시연 가이드를 복잡하게 만든다.

채택: 2번. 서버나 저장 스키마 변경 없이 현재 작업만 예제로 교체한다.

## Role Review

- business-analyst: IT 지식이 없는 현장 작업자가 앱 가치를 빠르게 확인하려면 결과 화면까지 바로 도달하는 경로가 필요하다.
- ui-ux-tester: 버튼은 `현재 작업` 헤더에 배치하고 모바일에서 48px 터치 타깃과 한 컬럼 배치를 유지한다.
- code-reviewer: 예제 로딩은 순수 유틸로 분리한다. 기존 커스텀 공간/박스, 파일 ID, 장치 ID, 마지막 백업 시각은 보존하고 반복 실행 시 예제 박스를 중복 누적하지 않는다.
- nextjs-developer: IndexedDB schema를 바꾸지 않고 기존 자동저장 흐름으로 저장되게 한다.

## Acceptance Criteria

- 현재 작업 영역에 `시연 예제 불러오기` 버튼이 보인다.
- 버튼을 누르면 `2.5톤반` preset과 예제 박스 3종이 현재 작업에 채워진다.
- 기존 내 공간과 저장된 박스는 삭제되지 않는다.
- 기존 결과와 추가 적재 기록은 지워져 예제 기준으로 다시 계산한다.
- 예제는 바로 `결과 만들기`가 가능한 입력 조건을 만든다.
- 같은 버튼을 반복해서 눌러도 예제 박스가 라이브러리에 중복 저장되지 않는다.
