# 현장 전달 전 점검

## PM 판단

V1 마무리 단계에서는 새 계산 기능보다 `결과를 만든 뒤 현장에 전달해도 되는가`를 사용자가 한눈에 확인하는 흐름이 중요하다. 현재 결과 화면에는 3D 보기, 쌓는 순서, 백업 파일 만들기가 각각 존재하지만, 현장 작업자는 이 행동들을 모두 끝냈는지 놓칠 수 있다.

검토한 방법:

1. 문서 체크리스트만 강화
   - 장점: 구현 위험이 없다.
   - 단점: 실제 앱 화면을 보는 작업자가 체크리스트를 따로 열어야 한다.
2. 결과 화면 하단에 `현장 전달 전 점검` 패널 추가
   - 장점: 최신 결과, 미적재/경고, 작업 지시서, 백업 파일을 한 화면에서 확인하고 기존 행동으로 바로 이동할 수 있다.
   - 단점: 결과 하단 정보가 하나 늘어나므로 모바일 layout 검증이 필요하다.
3. 별도 시연 완료 modal 추가
   - 장점: 시연 담당자에게 강하게 보인다.
   - 단점: 실제 현장 작업 중에는 불필요한 modal로 느껴질 수 있다.

채택: 2번. `시연`이라는 일회성 문구 대신 실제 운영에도 맞는 `현장 전달 전 점검`으로 표현한다.

## Role Review

- product-manager: V1 종료 기준인 결과 확인, 작업 지시서 전달, 백업 파일 생성 흐름을 한곳에 묶는다.
- business-analyst: IT 비숙련 현장 사용자는 계산 결과를 만든 뒤에도 작업 지시서와 백업 파일을 놓칠 수 있으므로, 결과 화면 안에서 다음 행동을 계속 보여줘야 한다.
- ui-designer: 기존 4단계 흐름과 결과 하단 grid를 유지하고, 새 패널은 sub-panel로만 추가해 화면 구조를 크게 흔들지 않는다.
- ui-ux-tester: 360px/390px 모바일에서 체크 항목과 액션 버튼이 한 컬럼으로 접히고 48px 터치 타깃을 유지해야 한다.
- code-reviewer: 새 저장 상태를 만들지 않고 `latestResult`, `resultFreshnessState`, 미적재/경고, 지시서 복사/저장 상태, `needsExport`만 읽는다.
- nextjs-developer: 순수 helper `createFieldHandoffChecklist`와 source-level layout test로 회귀를 고정한다.

## Acceptance Criteria

- 결과 화면 하단에 `현장 전달 전 점검` 패널이 표시된다.
- 결과가 없으면 `결과 만들기`가 첫 점검 항목으로 안내된다.
- 결과가 오래되면 `결과 다시 만들기`를 안내한다.
- 미적재 또는 경고가 있으면 현장 전달 전 확인 항목으로 표시한다.
- 작업 지시서를 복사하거나 저장하지 않았으면 `쌓는 순서 열기`를 안내한다.
- 백업 파일이 필요하면 `백업 파일 만들기`를 안내한다.
- 모든 항목이 준비되면 `현장 전달 준비됨`으로 표시한다.
- 모바일에서 점검 액션은 한 컬럼과 48px 터치 타깃을 유지한다.
