# 모바일 하단 주요 액션 접근성 보강

## PM 판단

V1 모바일/태블릿 시연에서 하단 고정 액션은 사용자가 현재 상태와 다음 행동을 계속 확인하는 안전장치다. 기존 구현은 시각적으로는 동작하지만, 상태 문구와 CTA 버튼의 관계가 접근성 속성으로 충분히 연결되어 있지 않아 보조기기나 작은 화면 검증 기준에서 추적하기 어려웠다.

이번 증분은 새 기능을 추가하지 않고, 기존 `sticky-mobile-actions`의 의미를 명확히 고정한다.

## 검토한 접근

1. 시각 스타일만 유지하고 브라우저 검증으로 확인한다.
   - 장점: 변경량이 없다.
   - 단점: 상태 문구와 버튼 관계가 코드 계약으로 남지 않는다.
2. 기존 하단 액션에 region, live status, described button을 추가한다.
   - 장점: UI 구조를 바꾸지 않고 현장 접근성과 테스트 가능성을 높인다.
   - 단점: 고정 ID 2개가 추가된다.
3. 모바일 워크플로우를 별도 상태머신과 bottom sheet로 재설계한다.
   - 장점: 장기 확장성은 크다.
   - 단점: V1 마무리 범위와 회귀 위험이 커진다.

채택: 2번. 현재 V1에는 레이아웃 변경보다 기존 행동의 의미와 검증성을 높이는 것이 더 안전하다. 하단 고정 액션은 모바일 전용으로 유지하고, 태블릿은 기존 본문 흐름을 사용한다.

## 적용 범위

- 하단 고정 액션 컨테이너를 `모바일 주요 작업` region으로 표시한다.
- 상태/도움말 텍스트 영역을 `role="status"`와 `aria-live="polite"`로 연결한다.
- 주요 CTA 버튼이 현재 상태와 막힌 이유를 `aria-describedby`로 참조한다.
- source-level layout test로 데스크톱/태블릿 숨김, 모바일 표시, 모바일 하단 여백 계약을 고정한다.

## 역할별 검토

- business-analyst: IT 비숙련 현장 사용자는 버튼 근처의 막힌 이유와 다음 행동을 함께 읽어야 한다.
- ui-ux-tester: 360px/390px 화면에서 하단 고정 영역은 콘텐츠를 가리지 않고, overlay가 열리면 숨겨져야 한다.
- code-reviewer: 저장, 결과 생성, 백업 동작은 변경하지 않고 접근성 속성과 테스트만 추가한다.
- nextjs-developer: 기존 React 상태와 CSS 구조를 유지하며 static export에 영향을 주지 않는다.

## 수용 기준

- 모바일 하단 액션은 region으로 식별된다.
- 상태 문구는 polite live region으로 갱신된다.
- CTA 버튼은 현재 상태와 helper 문구를 설명으로 참조한다.
- 데스크톱과 태블릿에서는 숨고 모바일에서만 고정 표시된다.
- `npm test`, `npx tsc --noEmit`, `npm run build`, 브라우저 검증을 통과한 뒤 커밋/푸시한다.
