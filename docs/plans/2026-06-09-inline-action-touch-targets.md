# 2026-06-09 Inline Action Touch Targets

## Goal

실행 전 검토 영역의 인라인 액션 버튼을 현장 모바일 터치 기준에 맞게 키운다. 320px 브라우저 계측에서 `백업 만들기`, `보호 강화` 버튼이 32px 높이로 확인되어, 장갑 착용이나 흔들리는 현장 환경에서 누르기 어려울 수 있다.

## Options Reviewed

1. 문구만 줄인다.
   - 장점: 레이아웃 변화가 작다.
   - 단점: 실제 터치 영역은 그대로 작다.
2. 모바일에서만 키운다.
   - 장점: 데스크톱 밀도를 유지한다.
   - 단점: 태블릿도 주 사용 환경인데 같은 버튼이 여전히 작게 남는다.
3. `.inline-action` 기본 높이를 44px, 모바일은 48px로 올린다.
   - 장점: 실행 전 검토 안의 `최신본 불러오기`, `백업 만들기`, `보호 강화`가 모두 일관된 현장 터치 타깃을 갖는다.
   - 단점: 체크리스트 행 높이가 조금 늘어난다.

## Decision

3번을 선택한다. WCAG 2.2 Target Size Minimum은 24px 이상을 기준으로 하지만, Package Tetris 기획서는 현장 주요 버튼과 모바일 CTA에 44~48px을 목표로 둔다. 실행 전 검토의 인라인 액션은 실제 복구/백업 행동이므로 더 큰 제품 기준을 적용한다.

## Role Review

- product-manager: V1 인수 기준 19~22번의 태블릿/모바일 터치 사용성을 보강한다.
- business-analyst: 현장 작업자가 저장 보호나 백업을 작은 텍스트 링크처럼 놓치지 않게 한다.
- ui-designer: 기존 체크리스트 구조는 유지하고 버튼 터치 영역과 줄바꿈만 키운다.
- ui-ux-tester: 320px, 360px, 390px, 768px, 1024px, 1440px에서 가로 넘침과 44px 미만 버튼을 다시 확인한다.
- code-reviewer: 기능 로직 변경 없이 CSS와 정적 레이아웃 테스트만 추가해 회귀 범위를 줄인다.

## Acceptance

- `.inline-action`은 기본 44px 이상 높이를 가진다.
- 모바일에서는 `.inline-action`이 48px 이상 높이를 가진다.
- 인라인 액션은 긴 문구에서도 줄바꿈을 허용하고 상위 행을 가로로 밀지 않는다.
