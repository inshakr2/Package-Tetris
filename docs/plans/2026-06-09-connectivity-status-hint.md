# 네트워크 상태 안내

## PM 판단

- V1은 서버 없이 IndexedDB와 백업 파일로 작업을 유지한다.
- 기획서에는 PWA 설치형 UX와 오프라인 재진입 안정화가 후속 항목으로 남아 있다.
- 이번 증분은 서비스워커 캐시나 설치 프롬프트를 만들지 않고, 브라우저가 인터넷 끊김을 감지했을 때 현장 작업자가 백업 필요성을 이해하도록 안내한다.

## 검토한 접근

1. 서비스워커 기반 오프라인 재진입 보장
   - 효과는 크지만 캐시 전략, 업데이트 전략, 정적 export 배포 검증이 필요해 이번 사이클 범위를 넘는다.
2. PWA 설치 프롬프트 UX
   - 설치형 경험을 강화하지만 오프라인 작업 안정성을 직접 해결하지 않는다.
3. 네트워크 상태 힌트
   - `navigator.onLine`과 `online/offline` 이벤트를 힌트로만 사용하고 기능을 막지 않는다.

채택: 3번.

## 적용 방향

- `navigator.onLine`은 정확한 인터넷 보장을 의미하지 않으므로 UI 문구는 `인터넷 끊김 감지`로 표현한다.
- 오프라인 감지 상태에서도 공간 선택, 박스 입력, 결과 확인 등 현재 화면 내 로컬 작업을 막지 않는다.
- 작업 데이터가 있으면 저장 상태 근처에 작은 amber pill을 표시하고, 저장 보호 패널에서 백업 파일 만들기를 권장한다.
- 페이지 전체 banner는 사용하지 않는다. 저장 충돌처럼 작업이 막힌 상태와 시각 우선순위를 분리하기 위해서다.

## 수용 기준

- 온라인 상태에서는 네트워크 상태 UI가 불필요하게 보이지 않는다.
- 오프라인 감지 상태에서는 `인터넷 끊김 · 백업 권장` pill이 보인다.
- 저장 보호 패널에는 `네트워크 상태` 행이 추가되고, 새로고침/창 닫기 전 백업 파일 만들기를 안내한다.
- 모바일 360px/390px에서 pill이 줄바꿈되어도 horizontal overflow가 없어야 한다.
- 네트워크 상태는 기능 차단 조건으로 사용하지 않는다.
