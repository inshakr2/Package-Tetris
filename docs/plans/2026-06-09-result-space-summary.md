# Result Space Summary Plan

**Goal:** 다중 공간 결과에서 현장 작업자가 각 공간에 어떤 박스가 들어갔는지 공간을 전환하기 전에 파악할 수 있게 한다.

## Product Manager Decision

채택 접근안은 `공간별 대표 박스 구성 요약`이다.

1. 공간 전환 후 확인만 유지
   - 사용자가 직접 Space 1, Space 2를 눌러야 내용을 알 수 있어 현장 확인 속도가 느리다.
2. 모든 박스 목록을 공간 버튼에 표시
   - 정보량이 많아져 모바일과 태블릿에서 공간 버튼이 과도하게 길어진다.
3. 대표 박스 구성 요약
   - 수량이 많은 유형부터 최대 2종을 보여주고 나머지는 `외 N종`으로 줄여, 빠른 파악과 모바일 밀도를 함께 만족한다.

## Scope

- 결과 좌측 Space 버튼에 `박스명 N개` 요약을 표시한다.
- 하단 `공간별 적재 결과`에도 같은 요약을 표시한다.
- 요약은 `PackedSpace`를 입력으로 받는 순수 함수에서 생성한다.
- 같은 템플릿이라도 결과 안의 표시 이름이 다르면 별도 구성으로 보여준다.
- 긴 박스명은 줄바꿈 가능하게 처리하고 가로 overflow를 만들지 않는다.

## Acceptance Criteria

- Space 버튼만 보고 각 공간의 대표 박스 구성을 알 수 있다.
- 하단 공간별 결과에서도 같은 정보를 다시 확인할 수 있다.
- 요약은 수량이 많은 유형 우선, 최대 2종 노출, 나머지 `외 N종` 규칙을 따른다.
- 빈 공간은 `적재 박스 없음`으로 표시한다.
- `npm test`, `npx tsc --noEmit`, `npm run build`가 통과한다.
