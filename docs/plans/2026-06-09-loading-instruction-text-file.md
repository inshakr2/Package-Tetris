# Loading Instruction Text File Plan

> V2 기준: 이 문서는 2026-06-09 V1 구현 이력입니다. V2에서는 배치상세/쌓는순서/작업지시서 산출물을 제거하고 3D와 공간 확인, 미적재/경고, 백업 상태 중심으로 대체했습니다.

## Context

브라우저나 기기 정책이 클립보드 쓰기를 막으면 `작업 순서 복사`는 실패 안내를 띄운다. 현장 작업자는 그래도 지시서를 전달해야 하므로, 같은 작업 순서 텍스트를 `.txt` 파일로 저장할 수 있는 보조 경로가 필요하다.

참고 자료:
- MDN URL `createObjectURL()`: Blob 같은 객체를 가리키는 object URL을 만들 수 있다.
- MDN URL `revokeObjectURL()`: 만든 object URL은 사용 후 해제해 메모리를 정리해야 한다.

## Compared Approaches

1. **텍스트 파일 저장 버튼**
   - 장점: 클립보드 권한과 무관하게 현장 지시서를 남길 수 있다.
   - 단점: 다운로드 폴더를 찾아야 하므로 복사보다 한 단계 무겁다.

2. **기존 백업 파일에 지시서 포함**
   - 장점: 데이터와 지시서가 함께 이동한다.
   - 단점: JSON 백업과 현장 작업 지시가 섞여 IT 비숙련 사용자에게 혼동을 줄 수 있다.

3. **인쇄/PDF 지시서**
   - 장점: 실제 출력물에 적합하다.
   - 단점: PDF 레이아웃과 출력 검증이 필요해 별도 큰 증분으로 두는 편이 안전하다.

선택: 1번. `작업 지시서 저장` 버튼을 `쌓는 순서` 패널에 추가하고, 기존 복사 텍스트와 동일한 내용을 `.txt` 파일로 내려받게 한다.

## Role Review

- product-manager: 클립보드 실패 리스크에 대한 직접 보완이며 서버 없이 가능하다.
- business-analyst: 현장 작업자는 메신저 복사가 막혀도 파일로 전달하거나 보관할 수 있어야 한다.
- ui-designer: 버튼은 기존 복사 버튼과 같은 액션 그룹에 두고 모바일에서는 세로로 접어 터치 실수를 줄인다.
- ui-ux-tester: 360px/390px에서 버튼 2개가 가로 넘침 없이 48px 터치 높이를 유지해야 한다.
- nextjs-developer: Blob URL을 만들고 사용 후 revoke하며, 기존 `stackingInstructionText`를 재사용한다.
- code-reviewer: 다운로드 후 object URL 정리, 실패 상태 안내, 백업 파일과의 문구 혼동을 확인한다.

## Acceptance Criteria

- `쌓는 순서`가 있으면 `작업 지시서 저장` 버튼이 표시된다.
- 버튼은 기존 복사 텍스트와 같은 내용을 `.txt` 파일로 저장한다.
- 다운로드 성공/실패 상태가 화면에 문장으로 표시된다.
- 모바일에서 복사/저장 버튼이 패널 밖으로 넘치지 않는다.
- 새 패키지 없이 브라우저 Blob URL 기반으로 동작한다.
