# 추가 결과 미리보기 인지 개선

> V2 기준: 이 문서는 2026-06-09 V1 구현 이력입니다. V2에서는 배치상세/쌓는순서/작업지시서 산출물을 제거하고 3D와 공간 확인, 미적재/경고, 백업 상태 중심으로 대체했습니다.

## PM 판단

- 추가 박스 시뮬레이션은 결과 뷰어의 3D/2D 화면 자체가 바뀌는 작업이다.
- 기존 `원본`/`추가 결과` 버튼은 툴바 안에 작게 들어가 새 버튼이 생겼다는 인지가 약했다.
- 기존 선택 해제 버튼은 실제로 선택된 박스 유형 강조를 해제하지만, 추가된 블록을 지우는 버튼처럼 오해될 수 있다.

## 검토한 방법

1. 선택 해제 버튼을 `강조 해제`로만 바꾼다.
   - 장점: 변경이 작다.
   - 단점: 추가 결과가 만들어졌다는 인지는 여전히 약하다.
2. 결과 뷰어 안에 `추가 결과 미리보기` 안내 바를 두고 비교/취소 버튼을 함께 제공한다.
   - 장점: 현재 화면이 원본인지 추가 결과인지 바로 알 수 있다.
   - 단점: 결과 뷰어 컨트롤 영역이 조금 커진다.
3. 원본 비교를 없애고 추가 결과만 보여준 뒤 되돌리기만 제공한다.
   - 장점: 화면은 단순하다.
   - 단점: 반영 전 비교라는 V1 수용 기준이 약해진다.

## 적용 방향

- 방법 2를 적용한다.
- 추가 가능 수량이 1개 이상이면 결과 뷰어 안에 `추가 결과 미리보기` 안내 바를 표시한다.
- 안내 바에는 `추가 결과 보기`, `원본 보기`, `미리보기 취소` 버튼을 둔다.
- 선택된 박스 유형 강조 해제 버튼은 `강조 해제`로 표시한다.
- `배치 상세`은 왼쪽, `쌓는 순서`는 오른쪽 순서를 유지하고 테스트로 고정한다.

## 역할별 검토

- business-analyst: 현장 작업자는 추가 결과를 반영하기 전에 원본과 추가 결과의 차이를 즉시 확인해야 한다.
- ui-designer: 비교 버튼은 3D 보기 근처의 안내 바 안에 두고, 취소 버튼은 같은 맥락 안에서 제공한다.
- ui-ux-tester: 390px와 1280px에서 안내 바, 비교 버튼, 모달 버튼 순서, 3D 캔버스가 가로 overflow 없이 표시되어야 한다.
- code-reviewer: 표시 상태와 선택 강조만 바꾸며 적재 엔진, IndexedDB 저장, 백업 파일 구조는 변경하지 않는다.

## 개발 수용 기준

- 추가 결과 미리보기 중 `추가 결과 보기`와 `원본 보기`가 눈에 띄는 안내 바 안에 표시된다.
- `미리보기 취소`는 추가 시뮬레이션 상태를 초기화한다.
- `강조 해제`는 선택된 박스 유형 강조만 해제한다.
- `배치 상세`과 `쌓는 순서` 버튼 순서는 바뀌지 않는다.
