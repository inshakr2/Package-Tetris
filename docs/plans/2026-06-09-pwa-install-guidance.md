# PWA 설치 안내

## Goal

현장 작업자가 Package Tetris를 태블릿이나 PC에서 앱처럼 다시 열 수 있도록 저장 상태 패널 안에 `앱 설치` 상태와 설치/안내 버튼을 추가한다.

## PM Decision

이번 증분은 서버, 저장소, 적재 엔진을 변경하지 않는다. 이미 구현된 manifest와 서비스워커 위에 설치 안내 UX만 보강한다.

검토한 방법은 다음과 같다.

1. 자동 설치 prompt만 제공
   - 장점: Chromium 계열에서는 가장 간단하다.
   - 단점: Safari 등 일부 브라우저에서 버튼이 동작하지 않아 현장 사용자가 막힐 수 있다.
2. 브라우저별 수동 안내만 제공
   - 장점: 모든 브라우저에서 설명이 가능하다.
   - 단점: 설치 prompt가 가능한 환경에서도 사용자가 메뉴를 직접 찾아야 한다.
3. 설치 가능 이벤트가 있으면 prompt, 없으면 수동 안내
   - 장점: 브라우저별 차이를 흡수하고 V1 프론트 단독 범위를 유지한다.
   - 단점: 설치 가능 이벤트는 브라우저가 제어하므로 항상 즉시 뜨지는 않는다.

선택: 3번. MDN 기준으로 `beforeinstallprompt` 이벤트가 발생하면 이벤트를 저장했다가 사용자 버튼 클릭 시 `prompt()`를 호출할 수 있다. 다만 브라우저 지원 차이가 있으므로 저장 패널에는 항상 홈 화면 추가 안내를 같이 제공한다.

## Role Review

- product-manager: 현장 시연 기기에서 앱 접근성을 높이는 저위험 증분이다.
- business-analyst: 설치는 데이터 보존과 다르므로 백업 파일 필요 문구를 계속 유지해야 한다.
- ui-designer: 저장 상태 패널의 안전장치 목록 안에 `앱 설치`를 추가하면 사용자가 찾기 쉽다.
- ui-ux-tester: 360px/390px에서 설치 버튼이 백업/저장 보호 버튼과 함께 줄바꿈되며 가로 넘침이 없어야 한다.
- code-reviewer: prompt 이벤트는 한 번만 사용할 수 있으므로 사용 후 ref를 비우고, 미지원 브라우저는 수동 안내로 돌아가야 한다.

## Acceptance

- 저장 상태 패널에 `앱 설치` 행이 표시된다.
- 설치 가능 이벤트가 있으면 `앱 설치` 버튼으로 브라우저 설치 확인 창을 연다.
- 설치 이벤트가 없거나 설치를 건너뛴 경우 Chrome/Edge/Safari 홈 화면 추가 안내를 보여준다.
- 설치됨 상태에서는 설치 버튼이 비활성화된다.
