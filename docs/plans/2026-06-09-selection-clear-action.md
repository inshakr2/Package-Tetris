# Selection Clear Action Implementation Plan

## Context

사용자는 박스 범례나 3D 블록을 눌러 특정 박스 유형을 강조할 수 있다. 이 기능은 요구사항의 "특정 블록 하나를 클릭할 경우 렌더링된 형태에서 해당 블록이 하이라이트" 조건과 연결된다. 현재 강조 해제는 같은 범례를 다시 누르거나 키보드 `Esc`를 써야 하므로, 모바일/태블릿 현장 작업자에게는 해제 방법이 충분히 드러나지 않는다.

## 3 Approaches

1. **범례 문구에 재클릭 안내만 추가**
   - 장점: 코드 변경이 가장 작다.
   - 단점: 터치 사용자가 행동 버튼으로 인식하기 어렵고, 3D 확대 화면에는 안내가 약하다.

2. **결과 뷰어 상태줄에 `강조 해제` 버튼 추가**
   - 장점: 선택 상태 바로 옆에 해제 행동이 보여서 현장 작업자가 이해하기 쉽다. 기존 선택 state와 `onClearSelection`을 재사용한다.
   - 단점: 상태줄 레이아웃을 모바일에서 다시 검증해야 한다.

3. **선택된 박스 상세 패널 추가**
   - 장점: 치수, 수량, 깨짐주의 등 상세 정보를 확장하기 쉽다.
   - 단점: 이번 증분에는 과하고 결과 영역이 다시 복잡해진다.

## Decision

2번을 적용한다. 선택된 박스 유형이 있을 때 2D 상태줄과 3D 상태 영역 근처에 `강조 해제` 버튼을 표시한다. 버튼은 기존 선택 해제 함수만 호출하고, 엔진/저장/백업/3D 렌더링 데이터 구조는 변경하지 않는다.

## Role Review

- product-manager: 요구사항의 박스 강조 기능을 실제 현장 조작으로 완성하는 작은 개선이다.
- business-analyst: IT 지식이 없는 작업자는 "다시 누르면 해제"보다 명시적 `강조 해제` 버튼을 더 쉽게 이해한다.
- ui-designer: 결과 상태 가까이에 한 가지 행동만 추가한다. 48px 터치 타깃을 유지하고 모바일에서는 줄바꿈으로 넘침을 피한다.
- ui-ux-tester: 360px/390px에서 버튼이 가로 스크롤을 만들지 않는지, 3D 확대/일반 상태가 유지되는지 확인한다.
- nextjs-developer: 기존 `selectedBlockTemplateId`, `clearSelectedBlockTemplate`, `onClearSelection`을 재사용한다.

## Test Plan

1. 선택된 박스가 있을 때 2D 상태줄에 `강조 해제` 버튼이 있고 `clearSelectedBlockTemplate`을 호출한다.
2. 선택된 박스가 있을 때 3D 캔버스 상태 아래에 `강조 해제` 버튼이 있고 `onClearSelection`을 호출한다.
3. CSS는 버튼 min-height 48px, 모바일 줄바꿈/폭 안정성을 유지한다.
4. `npm test`, `npx tsc --noEmit`, `npm run build`, 브라우저 360/390/768/1280 검증을 수행한다.
