# UI Designer Role Memory

## Product Framing

My Package Tetris는 물류/현장 작업자가 모바일과 태블릿에서 실수 없이 적재 결과를 확인하고 백업까지 마치는 도구다. UI 판단은 예쁜 시각화보다 입력 실수 방지, 현재 상태 이해, 다음 행동 명확성, 복구 가능성을 우선한다.

## Integrated UI Principles

- 현장 언어 우선: `IndexedDB`, `v0`, `JSON 이동본`, `WebGL` 같은 기술어는 보조 설명으로 내리고, 주 UI는 `이 기기 저장`, `백업 파일`, `계산 결과`, `3D 보기`처럼 말한다.
- 4개 세로 섹션 유지: `공간 선택 -> 박스 등록 -> 실행 전 확인 -> 결과 확인` 흐름은 현장 사용자에게 순차 작업으로 이해되기 좋다.
- 모바일은 다음 행동 1개만 강하게 보여준다.
- 3D는 검토 보조, 2D는 위치 확인 근거로 유지한다.
- 저장 보호는 경고가 아니라 작업 안전장치로 표현한다.

## Screen Guidance

- 상단 보조 문구는 `이 기기에 자동 저장 · 백업 파일로 이동 가능`처럼 표현한다.
- 섹션 제목은 작업 동사 중심으로 직접화한다.
- 실행 전 확인 섹션은 현장 체크리스트처럼 문제 없음, 주의, 실행 불가를 구분하고 `결과 만들기` 하나를 크게 둔다.
- 결과 영역은 `공간 목록 / 3D·2D 뷰어 / 박스 범례`를 유지한다.
- 모바일 3D는 이번 증분에서 인라인 버튼 중심으로 유지하고, 전체화면 `크게 보기`는 후속 작업으로 둔다.
- 저장 보호 패널은 `이 기기 저장`, `백업 파일`, `브라우저 보호`의 3행 구조를 유지한다.

## Accessibility And Copy Standards

- 현장 기준 터치 타깃은 최소 48px 높이를 목표로 한다.
- 버튼 간격은 최소 8px 이상 둔다.
- 위험 행동과 주요 행동은 붙여놓지 않는다.
- 포커스 표시는 명확한 외곽선과 충분한 대비를 유지한다.
- 문구는 `상태 + 행동` 순서로 쓴다.
- 주요 본문은 14px 미만을 피하고, 모바일 조작 문구는 16px 전후로 맞춘다.

## Increment Priorities

1. 상단/저장 관련 기술 문구를 현장 언어로 바꾼다.
2. 모바일 하단 고정 액션은 현재 상태와 다음 주요 행동을 보여준다.
3. 결과 뷰어 버튼은 모바일에서 한 줄 칩처럼 크게 누를 수 있게 한다.
4. 결과 대기 상태에 직접 CTA를 추가한다.
5. 저장 충돌/읽기 전용 상태 문구를 행동 중심으로 바꾼다.
