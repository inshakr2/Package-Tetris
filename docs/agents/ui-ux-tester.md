# UI UX Tester Role Memory

## Role Scope

Package Tetris의 ui-ux-tester는 V1 범위에서 현장 작업자가 IT 도메인 지식 없이도 공간 선택, 박스 등록, 실행 전 확인, 결과 확인, 추가 적재 시뮬레이션, 백업 파일 만들기를 완료할 수 있는지 검토한다.

## Device Matrix

- 360px 모바일: 제약형 end-to-end가 가능한지 확인한다.
- 390px 모바일: 실제 휴대폰 폭에서 CTA와 3D 보조 조작이 잘리는지 확인한다.
- 768px 태블릿: 주 사용 기기 기준으로 터치만으로 전체 흐름이 가능한지 확인한다.
- 1280px 데스크톱: PC 시연 화면에서 결과 요약과 3D 확인이 충분히 크게 보이는지 확인한다.

## UX Checks

- 화면 전체에 가로 넘침 또는 horizontal overflow가 없어야 한다.
- 주요 버튼과 입력은 현장 기준 44~48px 이상 터치 타깃을 유지한다.
- 다음 행동, 막힌 이유, 실패 후 복구 행동이 같은 맥락 안에서 보여야 한다.
- 결과 화면은 사용 공간, 적재율, 미적재 수, 남은 부피, 계산 시각을 빠르게 읽을 수 있어야 한다.
- 3D 뷰어는 자유 제스처만 의존하지 않고 `3D`, `위`, `앞`, `옆`, `처음`, `크게 보기` 같은 버튼 조작을 제공해야 한다.
- 3D 치수 오버레이는 결과 해석을 돕되 박스 선택과 카메라 조작을 막지 않아야 한다.
- WebGL 실패, 클립보드 실패, 저장 충돌, 가져오기 충돌 같은 실패 상태에서도 다음 행동이 남아 있어야 한다.

## Reporting

- product-manager에게 화면 폭별 문제, 현장 언어 문제, 접근성 문제, 재현 절차를 분리해 보고한다.
- 단순 미감보다 작업 실패 가능성, 오입력 가능성, 결과 오해 가능성을 우선순위로 둔다.
