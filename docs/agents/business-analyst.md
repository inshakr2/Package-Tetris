# Business Analyst Role Memory

## Role Scope

Package Tetris는 물류 현장 작업자가 파레트, 컨테이너, 2.5톤반 트럭 짐칸 같은 제한된 공간에 박스형 화물을 어떻게 쌓을지 빠르게 판단하도록 돕는 적재 의사결정 도구다.

핵심 사용자는 IT 도메인 지식이 거의 없는 현장 작업자다. 주 사용 환경은 태블릿이며, 모바일은 보조 환경이다. 사용자는 공간 선택, 박스 입력, 실행 전 확인, 결과 3D 확인, 추가 박스 시뮬레이션, 저장/백업까지 한 기기에서 끝낼 수 있어야 한다.

## User Context

- 사용자는 창고, 차량 근처, 적재 구역에서 사용한다.
- 사용자는 공간 종류, 박스 유형, 깨짐주의, 안전 여유, 적재 가능 크기 같은 개념을 처음 접할 수 있다.
- 화면 문구는 기술 용어보다 작업 언어를 우선한다.
- V1은 서버와 계정 없이 로컬 저장과 백업 파일로 운영한다.
- 태블릿은 터치만으로 전체 플로우가 완료되어야 한다.
- 모바일은 복잡한 편집과 자유 3D 조작보다 입력, 검토, 결과 요약, 백업까지 가능한 제약형 end-to-end를 목표로 한다.

## UX Principles

- 단계형 흐름을 기본으로 한다: 적재 공간 선택 -> 박스 등록 -> 실행 전 확인 -> 결과 확인.
- 각 단계에서 다음 행동과 막힌 이유를 텍스트로 알려준다.
- 결과 화면은 설명보다 확인을 우선한다.
- 3D는 전체 적재 감각을 주고, 위/앞/옆 보기는 위치 확인 근거로 유지한다.
- 저장 상태는 숨기지 않고 사용자가 이해할 수 있는 문장으로 보여준다.
- 오류 코드는 내부용이고 UI는 원인과 해결 행동을 말한다.
- 색상만으로 의미를 전달하지 않고 라벨, 아이콘, 텍스트를 함께 사용한다.

## V1 Priorities

1. 모바일/태블릿 숫자 입력은 큰 터치 타깃과 명확한 단위 `mm`, `개`를 제공한다.
2. 실행 전 확인은 총 박스 수, 총 부피, 공간 적재 가능 크기, 깨짐주의 규칙, 회전 가능 여부, 경고/오류를 한 번에 보여준다.
3. 결과 확인은 3D 기본 뷰, 공간 전환, 박스 유형 강조, 2D 보조 뷰 fallback을 유지한다.
4. 자동저장만 믿게 하지 않고 백업 파일 필요 상태와 마지막 백업 시각을 명확히 보여준다.
5. 주요 CTA는 언제 가능한지, 왜 막혔는지, 실패 후 무엇을 해야 하는지 함께 알려준다.

## Acceptance Criteria

- 태블릿에서 터치만으로 공간 선택, 박스 입력, 실행 전 확인, 결과 3D 확인, 저장/백업이 완료된다.
- 모바일 360px 기준에서 공간 선택, 박스 입력, 실행 전 확인, 결과 요약 확인, 백업 파일 만들기가 가능하다.
- 어떤 핵심 기능도 hover, 복잡한 키보드 단축키, 멀티터치 제스처만으로만 동작하지 않는다.
- 현장 사용자를 위한 문구는 IT 용어 설명이 아니라 작업 행동 중심으로 표현된다.
