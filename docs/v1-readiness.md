# Package Tetris V1 Readiness

## V1 완료 기준

Package Tetris V1은 프론트 단독 적재 시연 도구로 마감한다. V1의 완료 기준은 현장 작업자가 PC 또는 태블릿 브라우저에서 공간 선택, 박스 등록, 실행 전 확인, 결과 확인, 작업 지시서 전달, JSON 백업까지 끝낼 수 있는지다.

완료 범위:

- Next.js 정적 프론트엔드로 실행한다.
- 서버, 계정, API route, 외부 DB 없이 동작한다.
- 작업본은 같은 기기와 같은 브라우저의 IndexedDB에 자동저장한다.
- 작업 데이터는 JSON 백업 파일로 만들고 가져올 수 있다.
- 커스텀 공간과 커스텀 박스는 저장 후 재사용할 수 있다.
- 결과는 3D, 위, 앞, 옆 보기로 확인할 수 있다.
- 적재 결과는 공중 부양, 충돌, 경계 초과, 깨짐주의 정책 위반을 안전 게이트와 자동 테스트로 검증한다.
- 추가 박스 시뮬레이션과 안전 여유 조정 추천은 자동 적용이 아니라 현장 판단을 돕는 보조 기능으로 제공한다.
- 결과 화면은 현장 전달 전 점검으로 최신 결과, 미적재/경고, 작업 지시서, 백업 상태를 확인하게 한다.

## 마감 검증 명령

개발자 또는 시연 담당자는 V1 마감 확인 시 아래 명령을 실행한다.

```bash
npm run v1:verify
```

이 명령은 다음 순서로 실행된다.

1. `npm test`
2. `npx tsc --noEmit`
3. `npm run field:audit`
4. `npm run build`

중간에 실패하면 V1 마감 상태로 보지 않는다.

## 현장 파일럿 확인

아래 항목은 V1 코드 개발 잔여가 아니라 실제 운영 전 현장 파일럿에서 확인해야 하는 범위다.

- 실제 현장 데이터 기준 적재 정합성
- 현장 PC 기준 대량 박스 입력과 계산 시간
- 현장 태블릿과 모바일 터치 흐름
- 브라우저 저장소 삭제 또는 기기 교체 상황에서 JSON 백업 복구가 충분히 안내되는지 여부
- 작업 지시서 문구가 실제 작업자에게 충분한지 여부

현장 파일럿에서 위 항목이 실패하면 V1.1 또는 V2 요구사항으로 분리한다.

## V1 제외와 V2 후보

다음 항목은 V1 완료 기준에 포함하지 않는다.

- 서버 기반 여러 기기 자동 동기화
- 사용자 계정과 팀 권한
- 중앙 백업과 감사 이력
- WMS, TMS, ERP 연동
- 무게 중심, 하중 지지 한도, 동적 안정성 물리 모델
- 비정형 화물, 원통형 화물, 유연 포장
- 드래그 기반 수동 배치 편집

여러 기기 자동 이어하기, 팀 공유, 중앙 복구, 승인 이력이 필요해지는 시점이 서버 기반 V2 도입 트리거다.
